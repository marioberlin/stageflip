T-185: `video-mode` tool bundle + `bounce_to_aspect_ratios` tool.

Adds a 15th canonical bundle `video-mode` for StageFlip.Video-specific agent
tools. Today it ships one tool — `bounce_to_aspect_ratios` — which the
agent uses to plan the multi-aspect export fan-out that T-186 will execute
in parallel.

### `bounce_to_aspect_ratios`

```
Input:  { targets: AspectRatio[], basisPx?: number }
Output: { ok: true, basisPx, variants: { aspectRatio, label, width, height }[] }
      | { ok: false, reason: 'wrong_mode' | 'basis_out_of_range' }
```

- Accepts preset strings (`'16:9' | '9:16' | '1:1' | '4:5' | '21:9'`) or
  `{ kind: 'custom', w, h }`.
- `basisPx` (default 1080, range 360–4320) is the short-axis pixel basis;
  the long axis is derived from the aspect and both axes round up to
  even for H.264/H.265 codec compatibility.
- Returns `wrong_mode` when called against a non-video document.
- Does not mutate the document; T-186 reads the variants and runs N
  renders.
