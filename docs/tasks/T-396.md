# T-396 ŌĆö `AiGenerativeClip` `staticFallback` (curated example)

**Branch**: `task/T-396-ai-generative-clip-static-fallback`

## Goal

Ship the `staticFallback` for `family: 'ai-generative'`. When the network permission gate denies, the tenant policy refuses, or an export pipeline targets a static medium (MP4/PPTX/PDF), the mount-harness routes to a deterministic poster: an `ImageElement` rendering the `curatedExample` data URL supplied at authoring time. When `curatedExample` is absent, a single placeholder TextElement (empty text) is emitted.

T-396 is the `staticFallback` half of ADR-005 ┬¦D1's `AiGenerativeClip` row. After T-396 merges, `family: 'ai-generative'` is structurally complete and **all five ╬│-live family pairs ship** ŌĆö Phase 13 ╬│-live coverage is closed (modulo modality follow-ups deferred per T-395 D-T395-2).

T-396 registers `aiGenerativeStaticFallbackGenerator` against the **T-388a static-fallback generator registry** (already on `main`), not via a family-hardcoded harness branch.

## Out of scope

| Item | Why deferred |
|---|---|
| Live re-validation of the curated example at authoring time | Authoring concern. The schema accepts whatever the host supplies; no validation against a fresh provider call. |
| Localised placeholder copy | App-level i18n catalogue (CLAUDE.md ┬¦10). |
| Animated example reveal on the static path | A "static" fallback cannot animate. |
| Multi-resolution / responsive image set | Single image URL in v1. |
| External hosted curated-example URL with fetch-at-export | Determinism ŌĆö the data URL is the authoring-time bake; an external fetch at export time would intersect determinism. v1 enforces this at the schema layer (the `curatedExample.src` refine accepts ONLY `data:` URLs). | Curated examples for non-image modalities (audio / video) | T-395 D-T395-2 defers non-image modalities to a future task. When that task lands, `curatedExample` extends to a discriminated union `{ kind: 'image' | 'audio' | 'video', src, contentType }`. |

## Architectural decisions

### D-T396-1. Schema addition ŌĆö `curatedExample?` on `aiGenerativeClipPropsSchema`

T-395 shipped `aiGenerativeClipPropsSchema` with `posterFrame` but no curated-example field. T-396 adds:

```ts
// packages/schema/src/clips/interactive/ai-generative-props.ts (MODIFIED)
export const aiGenerativeClipPropsSchema = z.object({
  // ... existing T-395 fields ...
  /**
   * Captured curated example for the static fallback. Optional;
   * absent ŌåÆ an empty placeholder TextElement (host overrides via
   * app-level i18n).
   *
   * v1 accepts ONLY `data:` URLs (inline-baked at authoring time).
   * `http(s):` URLs are out of scope ŌĆö they would intersect the
   * determinism floor (the frame-runtime would have to fetch the
   * image at export time, which `check-determinism` forbids inside
   * `clips/**`). Widening the schema is a separate task gated on
   * a determinism-skill ruling for the external-image fetch path.
   * Same posture as T-394 D-T394-1 (web-embed posterImage).
   */
  curatedExample: z.object({
    src: z.string().refine(
      (s) => s.startsWith('data:'),
      'curatedExample.src must be a data: URL in v1; http(s) URLs are deferred per the out-of-scope table',
    ),
    contentType: z.enum(['image/png', 'image/jpeg', 'image/webp']).optional(),
  }).strict().optional(),
}).strict();
```

Browser-safe. Existing T-395 fixtures continue to validate (the field is optional).

### D-T396-2. Generator output ŌĆö single `ImageElement` rendering the curated example

`defaultAiGenerativeStaticFallback` returns:

1. When `curatedExample` is supplied: a single `ImageElement` filling the clip's bounds with `src = curatedExample.src`. The element's `transform` matches the canvas dimensions (`x: 0, y: 0, width, height`).
2. When `curatedExample` is absent: a single placeholder `TextElement` with empty text (host overrides via app-level i18n; same posture as T-390 / T-392 / T-394).

Layout is trivial ŌĆö no vertical-stacking discipline like T-390 / T-392 because there is only one element either way.

The generator casts `args.curatedExample.src as ImageElement['src']` to bypass the schema-level `AssetRef` regex ŌĆö same posture as T-388 (voice waveform poster) and T-394 (web-embed poster). The data URL is the authoring-time bake, not a storage-resolved asset reference.

### D-T396-3. Determinism

`defaultAiGenerativeStaticFallback` is subject to the broad ┬¦3 rule (no rAF, no random, no time, no fetch). Pure transformation over inputs. Same posture as T-388 / T-390 / T-392 / T-394.

The `curatedExample.src` is passed through verbatim ŌĆö the generator does NOT fetch the URL. v1 accepts ONLY `data:` URLs (the schema refine in D-T396-1 enforces this); the renderer decodes the data URL inline.

### D-T396-4. Telemetry

`ai-generative-clip.static-fallback.rendered` (attributes: `family`, `reason`, `width`, `height`, `hasExample` boolean, `exampleSrcLength` integer when example present ŌĆö NOT the URL itself; same privacy posture as T-394 D-T394-4).

The `exampleSrcLength` integer lets telemetry consumers distinguish a tiny placeholder from a substantial baked example without leaking the URL. Pin via test (AC #11).

### D-T396-5. Registration via T-388a registry

`packages/runtimes/interactive/src/clips/ai-generative/index.ts` gains a side-effect call:

```ts
import { staticFallbackGeneratorRegistry } from '../../static-fallback-registry.js';
import { aiGenerativeStaticFallbackGenerator } from './static-fallback.js';

interactiveClipRegistry.register('ai-generative', aiGenerativeClipFactory);  // existing T-395 line
staticFallbackGeneratorRegistry.register('ai-generative', aiGenerativeStaticFallbackGenerator);  // NEW
```

The harness's family-agnostic dispatch (T-388a) picks up the registration without further changes.

## Acceptance criteria

### Schema

1. `aiGenerativeClipPropsSchema` accepts a valid `curatedExample` (data URL).
2. `curatedExample: { src: 'https://cdn.example.com/example.png' }` ŌåÆ schema error (v1 rejects http(s) per the out-of-scope deferral; the refine fires).
3. `curatedExample: { src: 'relative/path.png' }` ŌåÆ schema error (refine rejects).
4. `curatedExample: { src: 'data:image/png;base64,xxx', extra: true }` ŌåÆ schema error (strict shape).
5. Existing T-395 fixtures (without `curatedExample`) continue to validate.

### Generator determinism

6. `defaultAiGenerativeStaticFallback({ width: 640, height: 360, curatedExample: { src: 'data:image/png;base64,abc' } })` returns the same Element[] byte-for-byte across two calls.
7. Different `curatedExample.src` ŌåÆ different markup.
8. Absent `curatedExample` ŌåÆ single placeholder TextElement (no ImageElement).

### Registry integration

9. After importing `clips/ai-generative`, `staticFallbackGeneratorRegistry.resolve('ai-generative')` returns the generator.
10. Authored `staticFallback` (non-empty) wins; the generator is still called with `reason: 'authored'` for telemetry (T-388a D-T388a-3 contract).

### Telemetry

11. `ai-generative-clip.static-fallback.rendered` emits with documented attribute shape; `hasExample` boolean and `exampleSrcLength` integer, NOT the URL itself.

### Bounds

12. The ImageElement's transform fits within the clip's bounds (no element extends past `width ├Ś height`). Trivial because the element fills the canvas exactly. Mirror T-394 AC #12.
