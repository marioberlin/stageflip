# T-383 — `ShaderClip` frontier-tier primitive

**Branch**: `task/T-383-shader-clip`

## Goal

Ship the first frontier-tier clip family: `ShaderClip`. A schema-validated `InteractiveClip` with `family: 'shader'`, registered in `interactiveClipRegistry`, mountable via the `ClipFactory` contract from T-306. Reuses the existing `@stageflip/runtimes-shader` rendering primitive (GLSL → canvas, deterministic uniforms, explicit-precision validator) for both `liveMount` and `staticFallback`-poster generation. Per ADR-005 §D2 the two paths must converge on the same pixels at any given frame.

T-383 is the first γ-core dispatch. It sets the **reuse-the-existing-runtime pattern** that T-384 (`ThreeSceneClip`) replicates against `@stageflip/runtimes-three`.

## Architectural decisions

### D-T383-1. Reuse `@stageflip/runtimes-shader` as the rendering primitive

Two implementation options were considered: greenfield a separate shader path inside `runtimes-interactive`, or wrap the existing runtime. **Chosen: wrap.**

The convergence guarantee in ADR-005 §D2 ("liveMount and staticFallback converge on the same frame output") is enforced by construction only when both paths share a single rendering core. Two implementations push convergence to code review — fragile, untestable. The existing `runtimes-shader` package is already frame-driven (no rAF inside it; GLSL validation, uniform binding, explicit-precision rule), and §3 code is a strict subset of what the interactive tier permits, so importing it from the interactive tier is safe in this direction (and only this direction — the interactive tier MUST NOT be imported from §3-bound code).

Sets the pattern: T-384 reuses `@stageflip/runtimes-three` analogously.

### D-T383-2. `runtimes-interactive` adds `@stageflip/runtimes-shader` as a workspace dependency

Add to `packages/runtimes/interactive/package.json` `dependencies`:

```json
"@stageflip/runtimes-shader": "workspace:*"
```

The factory imports `ShaderClipHost` directly. If `ShaderClipHost` is not currently exported from `runtimes-shader/src/index.ts`, T-383 includes a one-line export bump (no behavioral change; minor changeset on `@stageflip/runtimes-shader`).

### D-T383-3. Schema narrowing — typed `liveMount.props` for `family: 'shader'`

T-305 left `liveMount.props: z.record(z.unknown())` with an inline note that Phase γ extends with a discriminated union per family. T-383 adds the first narrowing:

```ts
// packages/schema/src/clips/interactive/shader-props.ts (NEW)
export const shaderClipPropsSchema = z.object({
  /** Fragment-shader source. Must declare explicit float precision (validated at mount time). */
  fragmentShader: z.string().min(1),
  /** Initial uniforms passed at mount time. Per-frame uniforms come from the updater. */
  initialUniforms: z.record(uniformValueSchema).default({}),
  /** Canvas dimensions in CSS pixels. */
  width: z.number().int().positive(),
  height: z.number().int().positive(),
  /** Frame at which `staticFallback` poster should be sampled. Default 0. */
  posterFrame: z.number().int().nonnegative().default(0),
}).strict();
```

The file is browser-safe (pure Zod). Lives in `packages/schema/src/clips/interactive/` to colocate per-family schemas.

`InteractiveClip` itself does not change shape; the discriminator is asserted at gate time by `check-preset-integrity` — when `family === 'shader'`, parse `liveMount.props` against `shaderClipPropsSchema`.

### D-T383-4. `staticFallback` posture — author-supplied poster as `ImageElement`

For T-383, the `staticFallback: Element[]` is a single `ImageElement` whose `src` is a pre-rendered poster image generated offline at `posterFrame` using the same GLSL. Poster generation is cluster-author tooling, not in T-383's leaf scope. T-383 tests convergence at runtime against a poster generated in-test from the same `ShaderClipHost`.

A future enhancement could introduce a `'shader-still'` element type that renders the GLSL deterministically inside frame-runtime, fully closing the loop schema-side. Deferred — `ImageElement` is sufficient and avoids enlarging the element schema.

### D-T383-5. Uniform-updater contract + `@uniformUpdater` tag

The factory accepts an optional uniform-updater function:

```ts
/** @uniformUpdater */
function shaderUniformsForFrame(args: {
  frame: number;
  fps: number;
  resolution: readonly [number, number];
  props: ShaderClipProps;
}): Record<string, UniformValue> { ... }
```

The function MUST take `frame: number` as a parameter and MUST NOT read `performance.now()` / `Date.now()` / `Math.random()` / timer APIs — enforced by T-309's `check-determinism` shader sub-rule (path-based detection on `packages/runtimes/interactive/src/clips/shader/**` plus decorator-based on the `@uniformUpdater` JSDoc tag).

Default updater (when caller does not supply): map `frame` / `fps` / `resolution` to `uFrame` / `uTime` / `uResolution` in the standard form already used by `@stageflip/runtimes-shader`.

### D-T383-6. `liveMount` driver — frame source via `MountContext` extension

Frame ticks are an interactive-tier-runtime-owned concern. T-383 extends `MountContext` (T-306 contract) with an optional `frameSource`:

```ts
export interface FrameSource {
  /** Subscribe to frame ticks. Returns unsubscribe. */
  subscribe(handler: (frame: number) => void): () => void;
  /** Synchronous current frame for first paint. */
  current(): number;
}

// packages/runtimes/interactive/src/contract.ts (MODIFIED)
export interface MountContext {
  // ... existing fields ...
  /** Optional frame source for frame-driven families (shader, three-scene). */
  frameSource?: FrameSource;
}
```

The extension is **backward-compatible** — optional field; T-306's existing contract tests continue to pass. Event-driven families (T-389 ai-chat, T-393 web-embed) ignore `frameSource`. Frame-driven families assert non-`undefined` at mount entry and throw a typed error otherwise.

Two implementations ship with T-383:

1. `RAFFrameSource` — wraps `requestAnimationFrame` for browser live-preview. Lives in `packages/runtimes/interactive/src/frame-source-raf.ts`. NOT subject to the shader sub-rule (it is the rAF shim, not a uniform updater).
2. `RecordModeFrameSource` — frame increments driven externally (e.g., by `renderer-cdp` in record mode). Provides `advance(N)` for tests + parity convergence assertions.

T-309's path-based determinism check applies to `clips/shader/**` only; the frame sources live one directory up at `runtimes-interactive/src/` and may use `requestAnimationFrame`.

### D-T383-7. Permission posture

`ShaderClip` declares `permissions: []` in `liveMount` per ADR-005 §D1 (no permissions). The permission shim short-circuits to grant; the pre-mount gate still runs (`tenantPolicy.canMount('shader')`).

### D-T383-8. Telemetry — emit lifecycle + GL diagnostics

Per `MountContext.emitTelemetry`, the factory emits:

- `shader-clip.mount.start` (attributes: `family: 'shader'`, fragment-shader length, dimensions)
- `shader-clip.mount.success` (attribute: time-to-first-paint in microseconds)
- `shader-clip.mount.failure` (attribute: failure reason — `'compile'` / `'link'` / `'context-loss'` / `'invalid-props'`)
- `shader-clip.dispose` on `MountHandle.dispose()`

Failure routes to `staticFallback` rendering by the mount-harness (T-306 already wires this; T-383 confirms the path runs).

### D-T383-9. `componentRef.module` value

The schema's `componentRefSchema` example was `@stageflip/runtimes-interactive/clips/shader#ShaderClip`. T-383 makes that real:

- Add a subpath export to `runtimes-interactive`'s `package.json`:

  ```json
  "./clips/shader": {
    "types": "./dist/clips/shader/index.d.ts",
    "import": "./dist/clips/shader/index.js",
    "require": "./dist/clips/shader/index.cjs"
  }
  ```

- The exported `ShaderClip` symbol is the React component name used at the resolution point. The `ClipFactory` is registered separately by side-effect at module-load time inside `clips/shader/index.ts`.

### D-T383-10. Convergence test in CI

Add `packages/runtimes/interactive/src/clips/shader/convergence.test.tsx`:

1. Render a fixed shader (e.g., a `uFrame`-driven gradient) via `ShaderClipHost` standalone at `frame=30`. Capture pixels.
2. Render the same shader via the `liveMount` factory using `RecordModeFrameSource.advance(30)`. Capture pixels.
3. Assert pixel-diff = 0 (bit-identical: same code, same uniforms, same frame). Document the epsilon choice (zero) in a test comment.

This test is the runtime expression of ADR-005 §D2 convergence — scoped to a unit test, distinct from the cluster-level parity harness from T-313.

## Acceptance criteria

### Schema

1. `shaderClipPropsSchema` parses a valid shader-props payload (fragment, dims, optional initial uniforms, optional posterFrame).
2. Empty fragment string → schema error.
3. Width / height of 0 or negative → schema error.
4. `check-preset-integrity` rejects an `InteractiveClip` with `family: 'shader'` whose `liveMount.props` does not parse `shaderClipPropsSchema`. Pin via fixture.
5. `check-preset-integrity` accepts a valid family-shader fixture.

### Factory

6. `interactiveClipRegistry.resolve('shader')` returns the factory after the side-effect import.
7. Re-importing throws `InteractiveClipFamilyAlreadyRegisteredError` (registry contract).
8. Factory mounts `ShaderClipHost`; first paint produced within 1 microtask of `frameSource.current()` returning.
9. `MountHandle.updateProps({ initialUniforms: { ... } })` propagates without re-mounting (verified via stable instance ref).
10. `MountHandle.dispose()` releases the GL context (verified via spy on `glContextFactory` teardown), unsubscribes from `frameSource`, and is idempotent (calling twice is a no-op).
11. `signal.abort()` triggers the same teardown path as `dispose()`.
12. Mount with `MountContext.frameSource === undefined` throws a typed `MissingFrameSourceError` for `family: 'shader'`. The error references ADR-005 §D2.

### Determinism + sub-rule integration

13. The default `@uniformUpdater` (in `clips/shader/uniforms.ts`) compiles `pnpm check-determinism` clean (broad rule + shader sub-rule).
14. The shader sub-rule's exported AST checker, invoked on a synthetic source containing `performance.now()` inside a `@uniformUpdater`-tagged function, returns a violation. Pin via Vitest unit test (no on-disk fixture that would break CI).

### Frame source

15. `RAFFrameSource.subscribe` emits at least 30 frames in 500 ms in the happy-dom test environment with a fake `requestAnimationFrame` implementation. Tolerate jitter; assert lower bound only.
16. `RecordModeFrameSource` emits exactly the frames an explicit `advance(N)` call dictates; deterministic; `subscribe` handlers receive each emitted frame in order.
17. Both implementations correctly unsubscribe on returned-callback invocation; no leaked handlers after disposal (verified by counter).

### Convergence (ADR-005 §D2)

18. Running the factory at `frame=30` produces a canvas whose pixel data is bit-identical to `ShaderClipHost` rendered standalone at `frame=30` with the same fragment + uniforms (D-T383-10). Pin via test; epsilon = 0.

### Permission posture

19. Mount of `family: 'shader'` does not trigger any permission prompt (verified via spy on `permission-shim`).

### Telemetry

20. Successful mount emits `shader-clip.mount.start` then `shader-clip.mount.success`; dispose emits `shader-clip.dispose`. Pin attribute shapes in tests.
21. Compile failure emits `shader-clip.mount.failure` with reason `'compile'` and falls back via the mount-harness path.

### Skills + drift

22. `skills/stageflip/runtimes/shader/SKILL.md` gains a §"Frontier-tier ShaderClip" with the reuse rationale, the `@uniformUpdater` convention, and the frame-source contract. `pnpm check-skill-drift` PASSES.
23. `skills/stageflip/concepts/runtimes/SKILL.md` references the new family entry.
24. `skills/stageflip/concepts/clip-elements/SKILL.md` documents the family-discriminator pattern.

## Escalation triggers (CLAUDE.md §6)

- `ShaderClipHost` requires API changes to be reusable from the interactive tier (e.g., assumptions about `frame-runtime-bridge` context that don't hold). Escalate; we may extract a lower-level primitive instead of wrapping.
- WebGL context creation in happy-dom is not faithful enough to validate convergence at the pixel level (zero-epsilon). Escalate; the convergence test may need a real-browser CI lane or a software-WebGL implementation.
- React 19 StrictMode double-mount semantics interact with WebGL context creation in a way that requires guard logic exceeding the factory's complexity budget. Escalate.
- The `componentRefSchema` regex in T-305 rejects the resolved module path. Escalate; ADR-003 may need an amendment.
- The `MountContext.frameSource` extension turns out NOT to be backward-compatible with T-306's existing contract tests (e.g., a `keyof` exhaustiveness check breaks). Escalate before merging — this contract change ripples to T-387+.
