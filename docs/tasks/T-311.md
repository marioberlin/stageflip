# T-311 — Type-design-consultant agent invocation tooling

**Branch**: `task/T-311-type-design-consultant-tooling`

## Goal

Stand up the **Orchestrator-side tooling** for invoking the type-design-consultant agent (per `skills/stageflip/agents/type-design-consultant/SKILL.md`). T-311 ships:

1. **`scripts/invoke-type-design-consultant.ts`** — CLI that assembles the consultant's inputs (cluster manifest + license-cleared registry + compass source) into a structured prompt, writes it to `reviews/type-design-consultant-cluster-<letter>.md`, and (optionally) invokes the agent.
2. **`reviews/`** directory — `.gitkeep` + README explaining the convention. Per-cluster review docs are committed here when the consultant's output lands.
3. **Cluster-batch lookup**: a `getClusterPresetManifest(cluster)` helper that consumes T-304's loader + T-307's font registry to produce the consultant's preset-manifest input.
4. **Invocation guards**: per the type-design-consultant SKILL.md §"Invocation", the script checks (a) cluster ∈ {A,B,D,F,G}; (b) batch review doesn't already exist (or rationale to re-run is supplied via `--reason=`); (c) compass file exists.

## Dependencies

- **Required reading**: `skills/stageflip/agents/type-design-consultant/SKILL.md` — the agent contract (already substantive, owner_task: T-311).
- T-304 (preset loader), T-307 (font registry) — merged.
- T-308 (`check-preset-integrity`) — merged. T-311's output (`reviews/type-design-consultant-cluster-<letter>.md`) is referenced by `signOff.typeDesign: 'signed:YYYY-MM-DD'` field in preset frontmatter; T-308 enforces that field but doesn't validate the link target exists. **Spec call**: T-311 ships the document; T-308 enforcement of "the linked review actually exists" is a future amendment.
- The actual AI agent invocation is deferred to operational tooling (the Orchestrator may dispatch via Claude API, manual CLI, or a future `Agent` tool integration). T-311 ships the **input assembly**, not the agent runtime.

## Architectural decisions

### D-T311-1. Script CLI surface

```
$ pnpm invoke-type-design-consultant --cluster=A [--reason=<text>] [--write-prompt] [--write-review-skeleton]
```

Flags:
- `--cluster=<A|B|D|F|G>` — required; uppercase letter or directory name (`news` → A).
- `--reason=<text>` — required if `reviews/type-design-consultant-cluster-<letter>.md` already exists (re-trigger guard).
- `--write-prompt` — write the assembled prompt to `reviews/.prompts/cluster-<letter>-<YYYY-MM-DD>.md` for inspection.
- `--write-review-skeleton` — write a placeholder `reviews/type-design-consultant-cluster-<letter>.md` with the structure SKILL.md §"Outputs" specifies, ready for the agent to fill in.

The script does NOT call any LLM API. It assembles the inputs, writes the artifacts, prints the next-step instructions for the operator.

### D-T311-2. Cluster mapping

Cluster letter ↔ directory:

| Letter | Directory | Bespoke fonts cluster-wide? |
|---|---|---|
| A | news | yes |
| B | sports | yes |
| C | weather | no — exempt |
| D | titles | yes |
| E | data | no — exempt |
| F | captions | yes |
| G | ctas | yes |
| H | ar | no — exempt |

Cluster C/E/H are EXEMPT — invoking with `--cluster=C` returns an error: "Cluster C does not require batch type-design review per ADR-004 §D4. Individual presets in this cluster may escalate one-off." Pin via test.

### D-T311-3. Preset-manifest format

For the assembled prompt, each preset in the cluster contributes:

```yaml
- id: cnn-classic
  preferredFont: { family: 'CNN Sans', license: 'proprietary-byo' }
  fallbackFont: { family: 'Inter Tight', weight: 700, license: 'ofl' }
  source: docs/compass_artifact.md#cluster-news--cnn-classic
  status: stub  # or 'substantive'
```

T-311 produces this YAML via `getClusterPresetManifest(cluster)`. Format-stable so the consultant agent's output can be diffed across re-reviews.

### D-T311-4. Output document skeleton

When `--write-review-skeleton` is set, T-311 writes:

```markdown
---
title: Type-design consultant — Cluster <letter> (<cluster-name>)
id: reviews/type-design-consultant-cluster-<letter>
reviewedAt: <YYYY-MM-DD>
clusterPresets: [<preset-id-list>]
signedOff: false
---

# Cluster <letter> — <Cluster Name> — Fallback review

## Cluster context
<auto-populated cluster description from skills/stageflip/presets/<dir>/SKILL.md>

## Per-preset reviews

### <preset-id>
**Bespoke**: <preferredFont.family> (<license>)
**Current fallback**: <fallbackFont.family or 'none'>

#### Three ranked candidates
1. _TBD by consultant_
2. _TBD by consultant_
3. _TBD by consultant_

#### Kerning / x-height / weight deltas
_TBD by consultant_

#### Rationale
_TBD by consultant_

#### Reference-frame recommendation
_TBD by consultant_

#### Final recommendation
_TBD by consultant_

(Repeated per preset...)

## Cross-preset coherence
_TBD by consultant_

## Escalations
_TBD by consultant_
```

The `signedOff: false` flips to `signedOff: true` (or a `signed:YYYY-MM-DD` line is added) when the user reviews the consultant output.

### D-T311-5. Re-trigger guard

Per SKILL.md §"Gate", re-review is required if any of:

- Any preset's `preferredFont` changes
- The license-cleared registry adds/removes a font this cluster depends on
- A fallback font's license posture changes

T-311 doesn't auto-detect these (operational); it just enforces "if the review file exists, you must supply `--reason=<text>` to overwrite". The reason is captured at the top of the regenerated review document.

### D-T311-6. Determinism + browser-bundle posture

Script is Node-only. Reads `@stageflip/schema/presets/node` (T-304/T-307 subpath). No browser-bundle implications.

## Acceptance criteria

### CLI surface

1. `pnpm invoke-type-design-consultant --cluster=A` exits 0; writes `reviews/type-design-consultant-cluster-A.md` (skeleton) by default.
2. `--cluster=C` exits non-zero with a "cluster does not require batch review per ADR-004 §D4" error.
3. **Re-trigger guard**: if `reviews/type-design-consultant-cluster-A.md` exists AND `--reason` is omitted → exit non-zero with a clear message. With `--reason="font-license-update"` → succeeds and writes a new skeleton, with the reason embedded.
4. `--write-prompt`: writes `reviews/.prompts/cluster-A-<YYYY-MM-DD>.md` containing the assembled prompt.
5. **Cluster name mapping**: `--cluster=A`, `--cluster=news`, and `--cluster=News` all resolve to cluster A.
6. **Unknown cluster**: `--cluster=Z` exits non-zero with cluster enumeration in the error.

### Preset manifest assembly

7. `getClusterPresetManifest('news')`: returns array of presets with `id`, `preferredFont`, `fallbackFont`, `source`, `status` per D-T311-3. Pin via fixture against the on-disk presets.
8. **Stable order**: presets sorted by `id` so the manifest is byte-stable across runs.
9. **Empty cluster**: a synthetic empty cluster directory returns an empty manifest with no errors.

### Skeleton output

10. Skeleton document includes per-preset placeholders for the 5 SKILL.md output sections (candidates, deltas, rationale, reference-frame, final recommendation).
11. Skeleton includes cluster-level "Cross-preset coherence" + "Escalations" sections.
12. Frontmatter includes `signedOff: false` and `clusterPresets: [<ids>]`.

### Backward compatibility

13. The full workspace test suite passes unchanged.
14. No existing scripts modified.

### Coverage + gates
