# T-387 — `VoiceClip` `liveMount` (Web Audio + MediaRecorder + transcript)

**Branch**: `task/T-387-voice-clip-live-mount`

## Goal

Ship the first γ-live family: `VoiceClip` `liveMount`. A schema-validated `InteractiveClip` with `family: 'voice'`, registered in `interactiveClipRegistry`, mountable via the T-306 `ClipFactory` contract. Captures microphone audio via `MediaRecorder`, streams an audio level signal via `Web Audio AnalyserNode`, and emits a transcript stream via `Web Speech API` (or a transcription-provider seam when the Web Speech API is unavailable).

T-387 is the **pattern-evaluation moment**. T-383 (shader) and T-384 (three-scene) wrap pre-existing §3 runtimes and converge `liveMount` ↔ `staticFallback` by construction. The remaining five γ-live families (voice, ai-chat, live-data, web-embed, ai-generative) do **not** converge and have **no §3 runtime to reuse** — they live entirely inside the interactive tier and are event-driven, not frame-driven. T-387's design choices set the second γ pattern.

T-387 ships `liveMount` only. **T-388 ships `staticFallback`** (waveform poster) as a separate S-sized task per the plan. Schema lands here; rendering of the fallback is T-388's scope.

## Architectural decisions

### D-T387-1. `VoiceClip` lives entirely in `runtimes-interactive/src/clips/voice/`

No new package; no §3 runtime to reuse (there is no `runtimes-voice` and there does not need to be). The voice clip is event-driven (microphone capture, MediaRecorder events, transcript callbacks), not frame-driven. The wrap-runtime pattern from T-383 / T-384 does not apply.

This is the **second γ pattern**: standalone interactive-tier clips with no §3 reuse, no frameSource dependency, and no convergence test. Pattern repeats for T-389 (ai-chat), T-391 (live-data), T-393 (web-embed), T-395 (ai-generative).

### D-T387-2. Schema narrowing — `voiceClipPropsSchema`

```ts
// packages/schema/src/clips/interactive/voice-props.ts (NEW)
export const voiceClipPropsSchema = z.object({
  /** Audio MIME type for MediaRecorder; browser-feature-detected fallback applies. */
  mimeType: z.string().default('audio/webm'),
  /** Maximum recording length in milliseconds; clip auto-stops at this duration. */
  maxDurationMs: z.number().int().positive().default(60_000),
  /** When true, the clip emits partial transcripts as the user speaks. */
  partialTranscripts: z.boolean().default(true),
  /** BCP-47 tag for transcription. Web Speech API `lang` setting. */
  language: z.string().default('en-US'),
  /** Frame at which `staticFallback` poster should be sampled (T-388 consumes). */
  posterFrame: z.number().int().nonnegative().default(0),
}).strict();
```

Browser-safe. Mirrors the discriminator pattern from T-383/T-384.

### D-T387-3. Permissions — `['mic']`; pre-prompt recommended in the skill

The clip declares `permissions: ['mic']`. The host application MUST install `permissionPrePromptHandler` on the mount-harness AND set `MountContext.permissionPrePrompt: true` for `family: 'voice'` mounts — pre-prompt UX is strongly recommended by the skill but not enforced at the contract layer (the user app may have its own permission UX via mic-pre-flow that subsumes ours).

If the permission gate denies, the mount-harness routes to `staticFallback` (T-388). Documented in the skill.

### D-T387-4. `liveMount` API surface

The factory mounts a small React tree:

```
<VoiceClipMount>
  <button data-action="record" />    // user-facing record toggle
  <canvas data-role="level-meter" /> // analyser-driven amplitude indicator
  <output data-role="transcript-live" /> // live transcript text node
</VoiceClipMount>
```

All visual structure is minimally styled and exposes data attributes for app-level overrides. Per CLAUDE.md §10, no English UI strings ship in the package — the host supplies button labels, transcript placeholder text via React props.

The factory exposes lifecycle callbacks via `MountHandle`:

```ts
interface VoiceClipMountHandle extends MountHandle {
  startRecording(): Promise<void>;
  stopRecording(): Promise<void>;
  /** Subscribe to transcript events. Returns unsubscribe. */
  onTranscript(handler: (event: TranscriptEvent) => void): () => void;
}

type TranscriptEvent =
  | { kind: 'partial'; text: string; timestampMs: number }
  | { kind: 'final'; text: string; timestampMs: number }
  | { kind: 'error'; message: string };
```

### D-T387-5. `TranscriptionProvider` interface — Web Speech API default + in-memory test impl

```ts
// packages/runtimes/interactive/src/clips/voice/transcription-provider.ts
export interface TranscriptionProvider {
  /** Start streaming transcription from the supplied MediaStream. */
  start(args: {
    stream: MediaStream;
    language: string;
    partial: boolean;
    onTranscript: (event: TranscriptEvent) => void;
  }): Promise<{ stop: () => void }>;
}
```

Two implementations ship with T-387:

1. `WebSpeechApiTranscriptionProvider` — wraps `window.SpeechRecognition` / `webkitSpeechRecognition`. Feature-detected; throws `WebSpeechApiUnavailableError` if neither is present.
2. `InMemoryTranscriptionProvider({ scripted })` — for tests; emits a scripted sequence of `TranscriptEvent` over time.

A cloud provider (OpenAI Whisper / AssemblyAI / Deepgram) is a future asset-gen task — Phase 14's provider-seam pattern (ADR-006) covers this; a sister ASR adapter at T-426a would land then. T-387 ships the seam, not the cloud adapters.

### D-T387-6. No convergence test

ADR-005 §D2 specifies convergence only for shader + three-scene. Voice has no rendered output to converge on. T-387 ships no `convergence.test.tsx`. Documented as out-of-scope in the file headers of the clip's index.

### D-T387-7. No `frameSource` dependency

Voice is event-driven. The factory does NOT throw `MissingFrameSourceError`; it ignores `MountContext.frameSource` if present. Pin via test (mounting with `frameSource: undefined` succeeds for voice).

### D-T387-8. Resource cleanup on dispose

`MountHandle.dispose()` MUST:

1. Stop the active `MediaRecorder` (`mediaRecorder.state === 'recording'` → `mediaRecorder.stop()`).
2. Stop every track on the captured `MediaStream` (`track.stop()` per track).
3. Disconnect the `AnalyserNode` from the audio graph.
4. Stop the `TranscriptionProvider` (calls the returned `stop` from `start`).
5. Unsubscribe all `onTranscript` handlers.
6. Be idempotent (calling twice is a no-op).

Resource leaks here are user-visible (browser shows a recording indicator that won't go away). This is the highest-attention area for testing.

### D-T387-9. Telemetry — lifecycle + recording + transcript events

Per `MountContext.emitTelemetry`:

- `voice-clip.mount.start` (attributes: `family: 'voice'`, language, partialTranscripts)
- `voice-clip.mount.success`
- `voice-clip.mount.failure` (reasons: `'permission-denied'`, `'web-audio-unavailable'`, `'web-speech-unavailable'`, `'media-recorder-unsupported-mime'`)
- `voice-clip.recording.started`
- `voice-clip.recording.stopped` (attribute: durationMs)
- `voice-clip.transcript.partial` (attribute: textLength) — text body NOT included in telemetry by default per privacy policy
- `voice-clip.transcript.final` (attribute: textLength)
- `voice-clip.dispose`

### D-T387-10. `componentRef.module` value + factory layout

- Subpath export `./clips/voice` added to `runtimes-interactive/package.json`.
- Factory at `clips/voice/index.ts`; side-effect registers `'voice'` family.
- Top-level functions per T-309a's tightening — same as T-384.

### D-T387-11. Pattern evaluation outcome — no abstraction extraction

T-383 and T-384 wrap §3 runtimes; T-387 does not. The shared parts between T-383/T-384/T-387 are:

1. Schema discriminator file at `packages/schema/src/clips/interactive/<family>-props.ts` (5 lines of structure)
2. Subpath export in `runtimes-interactive/package.json` (4 lines)
3. Side-effect registration `interactiveClipRegistry.register(family, factory)` (1 line)
4. Telemetry event naming `<family>-clip.<lifecycle>` (convention; no shared code)

None of these is "three similar lines" of meaningful logic — they are conventions enforced by the spec template, not duplicated implementations. **Per CLAUDE.md "three similar lines beat a premature abstraction": no shared abstraction is extracted at T-387.**

If T-389 / T-391 / T-393 / T-395 reveal genuine logic duplication (not convention), that's when extraction earns its place. T-387 documents this evaluation in the skill update so future Implementers understand the precedent.

## Acceptance criteria

### Schema

1. `voiceClipPropsSchema` parses a valid payload (mimeType, maxDurationMs, partialTranscripts, language, posterFrame).
2. `maxDurationMs` of 0 or negative → schema error.
3. Empty `language` string → schema error.
4. `check-preset-integrity` rejects `family: 'voice'` with bad props; accepts valid fixture.

### Factory + registry

5. `interactiveClipRegistry.resolve('voice')` returns the factory after side-effect import.
6. Re-importing throws `InteractiveClipFamilyAlreadyRegisteredError`.
7. Mount with `MountContext.frameSource: undefined` succeeds — no `MissingFrameSourceError` (D-T387-7).
8. Mount renders the React tree (record button + level meter + transcript live region).

### Recording lifecycle

9. `startRecording()` calls `getUserMedia({audio:true})` (the permission shim already vetted but the factory re-acquires its own stream per shim contract).
10. `startRecording()` instantiates `MediaRecorder` with the schema's `mimeType`; on unsupported MIME, falls back to default and emits `mount.failure` reason `'media-recorder-unsupported-mime'`. (Or, alternative behaviour: fall back transparently. Pick one and pin.)
11. `stopRecording()` after `startRecording()` resolves and the next `voice-clip.recording.stopped` telemetry event includes a non-zero `durationMs`.
12. Auto-stop after `maxDurationMs` is observed in telemetry.

### Transcription

13. With `WebSpeechApiTranscriptionProvider` injected and a stub `SpeechRecognition` that emits a `result` event, `onTranscript` handlers receive a `final` event with the matching text.
14. With `partialTranscripts: true`, partial events surface to handlers.
15. With `partialTranscripts: false`, partial events do NOT surface (final events still do).
16. Web Speech unavailable → mount fails with reason `'web-speech-unavailable'`; no telemetry leaks transcript text.
17. `InMemoryTranscriptionProvider({ scripted: [...] })` emits the scripted sequence in order; pin sequence determinism.

### Resource cleanup

18. `MountHandle.dispose()` stops the active `MediaRecorder`.
19. `MountHandle.dispose()` calls `track.stop()` on every track of the captured stream.
20. `MountHandle.dispose()` disconnects the analyser node.
21. `MountHandle.dispose()` calls the transcription provider's `stop()`.
22. `MountHandle.dispose()` is idempotent (second call is a no-op; pin via spy on resource teardown methods).
23. `signal.abort()` triggers the same teardown path.

### Telemetry

24. Each documented event (D-T387-9) is emitted with the documented attribute shape, observed via spy. Transcript text body is NOT included in telemetry attributes (privacy).

### Permission posture

25. Mount with `permissions: ['mic']` and a permissive permission shim succeeds; mount with denied gate routes to `staticFallback` per T-385's flow (verify the mount-harness sequence).

## Escalation triggers (CLAUDE.md §6)

- Web Speech API rejection without `result` events makes the test deterministic in happy-dom hard. Escalate; consider routing tests through `InMemoryTranscriptionProvider` exclusively.
- `MediaRecorder` MIME-type fallback semantics are inconsistent across browsers and the spec's auto-fallback (D-T387-2 default `'audio/webm'`) doesn't cover all targets. Escalate; the fallback chain may need to be explicit per target.
- Resource cleanup on `dispose` reveals a real-world leak path (e.g., the analyser node holds a strong reference that prevents GC). Escalate; the teardown sequence may need to be re-ordered.
- The transcription-provider seam interacts with T-415's asset-gen provider seam in a way that suggests early consolidation. Escalate; consider deferring T-387's seam shape until T-415's ADR lands (Phase 14).
