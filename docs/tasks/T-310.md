# T-310 — Extended `check-skill-drift` — presets + cluster skills

**Branch**: `task/T-310-skill-drift-presets`

## Goal

Extend `scripts/check-skill-drift.ts` to validate preset skills + cluster skills (per ADR-004 §D2 + the on-disk layout at `skills/stageflip/presets/{cluster}/{preset-id}.md`). The existing script's `link-integrity` + `tier-coverage` checks ALREADY scan all skills under `skills/stageflip/`, including presets. T-310 ADDS preset-specific drift detection that only T-304's loader can perform.

## Dependencies

- **Required reading**: ADR-004 §D2 (cluster co-location).
- T-304 (preset loader) — merged. T-310 calls `loadAllPresets` to build a deeper view than the existing `loadSkillTree`.
- T-307 (font-license registry) — merged. T-310 surfaces font-vocabulary drift.
- T-308 (`check-preset-integrity`) — merged. **T-310 is NOT a duplicate** — T-308 enforces ADR-004 §D6 invariants on individual presets at PR time. T-310 enforces **TREE-LEVEL** invariants on the cluster + preset coverage:
  - Every cluster directory has a `SKILL.md` (cluster skill).
  - Every cluster skill's `tier: cluster` matches its directory location.
  - No orphaned preset files (preset frontmatter `cluster` field matches its directory).
  - Cluster skill's frontmatter `id` matches its filesystem location.

## Architectural decisions

### D-T310-1. Two new tree-level checks

Append to the existing check pipeline:

1. **`preset-cluster-coverage`**: walks `skills/stageflip/presets/`. For each subdirectory, verifies a `SKILL.md` exists. For each `*.md` (excluding `SKILL.md`), parses frontmatter via T-304 loader and verifies `cluster` field matches the parent directory name.
2. **`preset-id-coherence`**: verifies preset frontmatter `id` is consistent with filesystem location (the convention is `id` matches the filename without `.md` extension, OR explicitly differs and the cluster skill lists it).

### D-T310-2. Implementation

Modular: T-310 ADDS a new function `presetCoverageCheck(tree)` and a new function `presetIdCoherenceCheck(tree)` to `check-skill-drift.ts`. Each returns a `CheckResult` matching the existing shape.

Both functions consume T-304's loader output (`loadAllPresets`) — invoked from inside the script.

### D-T310-3. Output format

Append to existing output:

```
check-skill-drift [link-integrity]: PASS
check-skill-drift [tier-coverage]: PASS
check-skill-drift [preset-cluster-coverage]: PASS (8 clusters, 50 presets)
check-skill-drift [preset-id-coherence]: PASS

check-skill-drift: PASS
```

### D-T310-4. Determinism + browser-bundle posture

Script is Node-only. Existing patterns from `check-skill-drift.ts` apply.

## Acceptance criteria

### `preset-cluster-coverage` (AC #1–#4)

1. Walks `skills/stageflip/presets/`, identifies the 8 cluster directories.
2. **Every cluster has a `SKILL.md`**: a synthetic test where one cluster dir lacks `SKILL.md` fails the check with the cluster name in the error.
3. **Preset's `cluster` matches directory**: a synthetic preset at `presets/news/foo.md` with `cluster: 'sports'` in frontmatter fails with the mismatch.
4. **PASS at HEAD**: against the actual on-disk presets, this check PASSES.

### `preset-id-coherence` (AC #5–#7)

5. **`id` matches filename**: a preset at `presets/news/cnn-classic.md` with frontmatter `id: 'cnn-classic'` PASSES; with `id: 'mismatched'` FAILS.
6. **Cluster skill `id`**: the cluster `SKILL.md` at `presets/news/SKILL.md` must have frontmatter `id: 'skills/stageflip/presets/news'`.
7. **PASS at HEAD**: against actual on-disk presets, this check PASSES.

### Integration

8. `pnpm check-skill-drift` runs all 4 checks (existing 2 + new 2). Exits 0 on PASS.
9. Output format matches the existing `[name]: PASS/FAIL` pattern.
10. **Aggregates**: violations across all 4 checks surface in a single report (no fail-on-first).
