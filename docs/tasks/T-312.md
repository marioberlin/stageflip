# T-312 — CLAUDE.md §6 amendment verification

**Branch**: `task/T-312-claude-md-section-6-verify`

## Goal

Verify that **CLAUDE.md §6 contains the two P13-specific escalation paths** that were added during the P13 scaffold (per `implementation-plan.md` v1.17 entry), and add a machine-checkable test that asserts the language doesn't drift.

The two escalation paths (already on `main`):

1. *"Implementer and Reviewer disagree on whether a preset matches its compass source and reference-frame parity does not resolve it. Escalate with both candidate frames and both interpretations; Orchestrator approves or routes to the user."*
2. *"Type-design consultant returns 'no adequate fallback' for a bespoke font referenced by a preset in clusters A/B/D/F/G (see ADR-004 §D4)."*

T-312 ships:

1. A test (`scripts/check-claude-md.test.ts` or extending an existing harness) that parses CLAUDE.md and asserts both lines are present in §6.
2. (Bonus, if cheap) a check-script `scripts/check-claude-md.ts` that runs in CI and fails if the lines are deleted/edited.

## Dependencies

- None functionally. CLAUDE.md is a top-level doc; T-312 verifies its content.
- Prior P13 scaffold (v1.17 plan amendment) — already landed; T-312 proves it.

## Out of scope

| Item | Why deferred |

## Architectural decisions

### D-T312-1. Approach: vitest test, not a separate script

Adding a script + new `pnpm` command for ONE verification is overkill. T-312 ships a small vitest test that:

1. Reads `CLAUDE.md` from disk via `readFileSync`.
2. Asserts the two specific bullet-point strings (or stable substring matches) are present in §6.
3. Asserts §6 still exists (the broader header didn't disappear).

The test runs as part of `pnpm test`. Failure surfaces clearly: "CLAUDE.md §6 missing escalation path: 'preset matches its compass source'..."

### D-T312-2. Test location

Place at `scripts/check-claude-md.test.ts`. Path mirrors the script-test convention from T-308/T-310. Could equally live in a top-level `tests/` directory; placing in `scripts/` keeps test colocation patterns consistent with recent precedents.

### D-T312-3. Substring matching, not regex

The escalation-path bullets use specific phrasing that future drafts of CLAUDE.md should preserve. Test asserts via `.includes(...)` on stable substrings:

- `"preset matches its compass source"` (compass dispute path)
- `"no adequate fallback"` (type-design path)
- `"clusters A/B/D/F/G"` (cluster scope per ADR-004 §D4)

These three substrings together identify both lines. If any is missing, the test fails with a clear message.

1. **Test exists**: `scripts/check-claude-md.test.ts` is added; runs as part of `pnpm test`.
2. **Compass-dispute path present**: a test asserts CLAUDE.md `§6` contains `"preset matches its compass source"`.

4. **Cluster scope reference present**: a test asserts CLAUDE.md `§6` contains `"clusters A/B/D/F/G"` (or equivalent stable phrasing).
5. **§6 header itself exists**: a test asserts the heading `"## 6. Escalation"` is present.
6. **Test fails meaningfully**: when a substring is removed (synthetic mutation in a tmp file), the test surfaces a clear error message naming the missing path.
