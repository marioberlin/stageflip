# T-304 — `packages/schema/src/presets/` preset schema primitive

**Branch**: `task/T-304-preset-schema-primitive`

## Goal

Stand up the preset schema primitive — the **loader + validator + frontmatter parser** that turns the SKILL.md files at `skills/stageflip/presets/{cluster}/{preset-id}.md` into typed objects consumable by downstream code (semantic router, render-time selection, parity harness).

Per **ADR-004 §D1** (preset = SKILL.md with YAML frontmatter), the source of truth is a directory tree of markdown files. T-304 ships:

1. **`presetFrontmatterSchema`** — Zod schema for the required + optional frontmatter fields.
2. **`presetSchema`** — Zod schema combining frontmatter + parsed body sections (Visual tokens / Typography / Animation / Rules / Acceptance / References).
3. **`loadPreset(filePath)`** / **`loadCluster(clusterPath)`** / **`loadAllPresets(rootPath)`** — three-tier loader API.
4. **`presetRegistry`** — in-memory index keyed by `{cluster, id}` for O(1) lookup.
5. **Strict validation** — unknown frontmatter fields throw with a clear path; missing required fields throw at parse time.

T-304 does NOT implement the font-license registry (T-307) or the `check-preset-integrity` CI gate (T-308) — both reference `packages/schema/src/presets/` but ship as separate tasks.

## Dependencies

- **Required reading**: ADR-004 (preset system), ADR-003 (interactive runtime tier — `permissions` field), ADR-005 (frontier clip catalogue — referenced from the `clipKind` field).
- T-301/T-302/T-303 (the three ADRs) — already on `main` (ratified 2026-04-25).
- The 8 cluster preset directories already exist at `skills/stageflip/presets/{ar,captions,ctas,data,news,sports,titles,weather}/`. T-304 reads them; it does NOT modify them.

## Architectural decisions

### D-T304-1. Frontmatter library: `gray-matter` (MIT)

`gray-matter` is the de-facto Node library for YAML frontmatter + body parsing. MIT, on whitelist. ~30 KB minified. Already a transitive dep of common tooling; verify on whitelist.

Alternative: hand-roll. Rejected — frontmatter parsing has gnarly edge cases (multiline strings, escape sequences, comment handling) and gray-matter is battle-tested.

### D-T304-2. Eager + lazy loader API

- **`loadPreset(filePath): Preset`** — synchronous, parses one file, throws on validation failure.
- **`loadCluster(clusterPath): { skill: ClusterSkill; presets: Preset[] }`** — reads the cluster's `SKILL.md` AND every other `.md` in the directory, returns aggregated.
- **`loadAllPresets(rootPath): PresetRegistry`** — walks `skills/stageflip/presets/*/` and builds the registry.

`loadAllPresets` is cached internally (memoized); the cache is keyed by `rootPath`. Reset via `presetRegistry.reset()` (test-only export).

### D-T304-3. Frontmatter shape (per ADR-004 D1)

```ts
export const presetFrontmatterSchema = z.object({
  id: z.string().regex(/^[a-z0-9][a-z0-9-]*[a-z0-9]$/),
  cluster: z.enum(['news', 'sports', 'weather', 'titles', 'data', 'captions', 'ctas', 'ar']),
  clipKind: z.string().min(1),
  source: z.string().min(1),
  status: z.enum(['stub', 'substantive']),
  preferredFont: z.object({
    family: z.string().min(1),
    // license: vocabulary deliberately NOT enumerated in T-304. T-307
    // (font-license registry) owns the canonical license vocabulary per
    // ADR-004 §D3. Stub vocabulary already includes 'commercial-byo',
    // 'platform-byo', 'na', composite expressions ('apache-2.0 + ofl'),
    // etc. — freezing an enum here forces premature decisions T-307 will
    // revisit. T-304 validates SHAPE (string present, non-empty); T-307
    // validates VOCABULARY (canonicalize composites, enforce whitelist).
    license: z.string().min(1),
  }),
  fallbackFont: z.object({
    family: z.string().min(1),
    weight: z.number().int().positive(),
    license: z.string().min(1),  // same rationale as preferredFont.license
  }).optional(),
  permissions: z.array(z.enum(['network', 'mic', 'camera', 'geolocation'])).default([]),
  signOff: z.object({
    parityFixture: z.string().regex(/^(pending-user-review|signed:\d{4}-\d{2}-\d{2}|na)$/),
    typeDesign: z.string().regex(/^(pending-cluster-batch|signed:\d{4}-\d{2}-\d{2}|na)$/),
  }),
}).strict();
```

`.strict()` rejects unknown FIELDS at the top level. This is a security primitive — typos in frontmatter field names must fail loud, not silently default. License VALUES are intentionally permissive in T-304 (validated as non-empty strings); T-307 owns the canonical license enum + composite-expression handling.

**Orchestrator amendment 2026-04-28** (Option C): the original spec froze license enums (`'ofl' | 'proprietary-byo' | …`) but the on-disk preset stubs already use vocabulary outside that enum (`commercial-byo`, `platform-byo`, `na`, `apache-2.0 + ofl`, etc.). Per ADR-004 §D3 the font-license registry is T-307's domain; T-304 widens the schema to `z.string().min(1)` with a TODO link to T-307. Top-level `.strict()` is preserved.

### D-T304-4. Body section extraction (best-effort)

The body is split on H2 headers (`## Visual tokens`, `## Typography`, etc.). Each section becomes a string field on the parsed object:

```ts
export interface PresetBody {
  visualTokens: string;       // section content as raw markdown
  typography: string;
  animation: string;
  rules: string;
  acceptance: string;         // parity acceptance criteria
  references: string;
  // unknown sections preserved here for forward-compat
  unknown: Record<string, string>;
}
```

Body sections are **not** strictly validated in T-304 (spec § Out of scope). Downstream tasks tighten section schemas as needed.

### D-T304-5. Cluster skill shape

The cluster `SKILL.md` (e.g., `skills/stageflip/presets/news/SKILL.md`) has different frontmatter:

```ts
export const clusterSkillFrontmatterSchema = z.object({
  title: z.string().min(1),
  id: z.string().regex(/^skills\/stageflip\/presets\/[a-z]+$/),
  tier: z.literal('cluster'),
  status: z.enum(['stub', 'substantive']),
  last_updated: z.string().regex(/^\d{4}-\d{2}-\d{2}$/),
  owner_task: z.string().regex(/^T-\d+/),
  related: z.array(z.string()).default([]),
}).strict();
```

`loadCluster` parses this with `clusterSkillFrontmatterSchema`; preset files in the same dir parse with `presetFrontmatterSchema`. The loader distinguishes by the `tier: 'cluster'` discriminator OR by filename === 'SKILL.md'.

### D-T304-6. Error semantics

- **Validation errors** throw `PresetValidationError` (extends `Error`) with `filePath`, `field`, and `zodIssues` populated.
- **File-not-found** throws standard Node `ENOENT` (don't wrap; consumers catch via `code === 'ENOENT'`).
- **YAML parse errors** throw `PresetParseError` with `filePath` + the underlying gray-matter error wrapped.

`loadAllPresets` AGGREGATES errors — if any preset fails to parse, ALL parse failures are reported in a single `PresetRegistryLoadError` with an array of issues. This is faster developer feedback than fail-on-first.

### D-T304-7. Determinism posture

`packages/schema/src/presets/**` is NOT clip / runtime code. The loader uses `fs.readFileSync` (synchronous) + `path.join` — no time-based or random behavior. Determinism scan rules don't apply.

The OUTPUT (`Preset` objects) is canonical: same input file → same parsed object across builds. Pin via test that hashes the parsed registry.

## Acceptance criteria

Each gets a Vitest test, written first and failing.

### Frontmatter (`frontmatter.ts`)

1. `presetFrontmatterSchema.parse({ ...valid })` accepts a complete frontmatter record per D-T304-3.
2. **Unknown field**: `presetFrontmatterSchema.parse({ ...valid, extraField: 'x' })` throws (`.strict()` enforced).
3. **Missing required**: omitting `id` / `cluster` / `clipKind` / `source` / `status` / `preferredFont` / `signOff` throws with a clear `field` in the error.
4. **Invalid `id` format**: `'CNN_classic'` or `'cnn classic'` (uppercase / space) throws; `'cnn-classic'` parses.
5. **Cluster enum**: only the 8 documented clusters accepted. (License fields are deliberately NOT enumerated — see D-T304-3 amendment; T-307 owns the canonical license vocabulary.)
6. **`signOff` regex**: `'pending-user-review'`, `'signed:2026-04-25'`, `'na'` accept; arbitrary strings reject.
7. **`fallbackFont` optional**: a preset without `fallbackFont` parses (clusters that don't cite bespoke fonts).
8. **`permissions` defaults to `[]`** when omitted.

### Body extraction (`body.ts`)

9. A markdown body with the canonical 6 sections parses into 6 string fields. Pin via fixture.
10. **Section-order tolerance**: sections in any order produce the same parsed shape (order is structural, not semantic).
11. **Unknown sections**: extra H2s like `## Notes` land in `body.unknown['Notes']`. Pin via test.
12. **Missing sections**: a body missing one of the 6 canonical sections leaves the corresponding string EMPTY (NOT undefined; downstream consumers check `.length === 0` for "missing"). Pin via test.

### Loader (`loader.ts`)

13. `loadPreset(filePath)`: reads + parses + validates. Returns `Preset` object on success.
14. `loadPreset` for a non-existent file: throws `ENOENT` (standard Node error; not wrapped).
15. `loadPreset` for invalid YAML: throws `PresetParseError`.
16. `loadPreset` for valid YAML but bad frontmatter shape: throws `PresetValidationError` with `filePath` populated.
17. `loadCluster(clusterPath)`: reads the cluster's `SKILL.md` (cluster-skill schema) AND every `*.md` in the dir (preset schema). Returns `{ skill, presets }`.
18. `loadCluster` skips files that aren't `.md` (e.g., README.txt). Pin via test.
19. `loadCluster` throws when `SKILL.md` is missing.
20. `loadAllPresets(rootPath)`: walks `rootPath/*/` (one level deep), invokes `loadCluster` for each. Returns `PresetRegistry`.
21. **`loadAllPresets` aggregates errors**: if 3 presets fail to parse, all 3 issues appear in a single `PresetRegistryLoadError`. Pin via fixture.
22. **`loadAllPresets` is memoized**: second call with the same `rootPath` returns the cached registry (reference equality acceptable).

### Registry (`registry.ts`)

23. `presetRegistry.get(cluster, id)`: O(1) lookup; returns `Preset | undefined`.
24. `presetRegistry.list(cluster?)`: returns all presets in a cluster, or all presets across clusters if cluster omitted.
25. `presetRegistry.clusters()`: returns sorted list of cluster names that loaded successfully.
26. `presetRegistry.reset()`: clears the cache (test-only API; production code should not call this).

### Errors (`errors.ts`)

27. `PresetValidationError` extends `Error` and exposes `filePath`, `field?`, `zodIssues`.
28. `PresetParseError` extends `Error` and exposes `filePath`, `cause` (the original gray-matter error).
29. `PresetRegistryLoadError` extends `Error` and exposes `issues: Array<{ filePath, error }>`.

### Determinism

30. **Output canonicality**: `loadAllPresets(rootPath)` produces the same `JSON.stringify(registry.list())` across 100 invocations. Pin via test.

### Backward compatibility

31. The full `@stageflip/schema` test suite passes unchanged.
32. **No existing skill file is modified** — the loader reads the existing 8 cluster directories and 50+ preset stubs as-is.

### Coverage + gates

33. Coverage on `packages/schema/src/presets/frontmatter.ts` ≥95% (security primitive).
34. Coverage on `packages/schema/src/presets/loader.ts` ≥90%.
35. Coverage on `packages/schema/src/presets/registry.ts` ≥90%.

## Escalation triggers (CLAUDE.md §6)

- An existing preset stub at `skills/stageflip/presets/*/` fails the schema (AC #37). Don't silently fix the stub; escalate. The stubs were authored against ADR-004 D1; if the schema diverges, the ADR or the stubs need amendment, not a silent fix.
- `gray-matter` introduces a transitive dep with a non-whitelisted license. Escalate; we may need to swap to a different parser (`yaml` + manual body split is the lightweight alternative).
- The cluster-skill discriminator (filename === 'SKILL.md' OR `tier: 'cluster'`) doesn't reliably distinguish cluster skills from preset skills. Escalate; we may need an explicit `kind: 'preset' | 'cluster'` discriminator field added to ADR-004 D1.
- Body-section H2 split is ambiguous (e.g., a preset uses `### Visual tokens` instead of `## Visual tokens`). Escalate; the section-level schema needs to either tolerate H3s or pin H2-only.
- Frontmatter `signOff.parityFixture` value `'pending-user-review'` clashes with a preset author who wrote `'pending-product-owner'`. Escalate; the value enum is part of the spec contract.
