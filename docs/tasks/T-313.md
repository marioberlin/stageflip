# T-313 — Parity-fixture pipeline + sign-off workflow

**Branch**: `task/T-313-parity-fixture-pipeline`

## Goal

Stand up the **parity-fixture auto-generation pipeline + sign-off workflow** per ADR-004 §D5. T-313 ships:

1. **`scripts/generate-preset-parity-fixture.ts`** — CLI that takes a preset id, renders its canonical configuration via the existing parity infra (`packages/testing/`), and produces a parity-fixture artifact + a goldens directory at the standard path.
2. **Sign-off workflow doc** — `docs/ops/parity-fixture-signoff.md` codifying the ADR-004 §D5 product-owner-per-cluster process.
3. **`signOff.parityFixture: 'signed:YYYY-MM-DD'`** integration — when the user signs off a fixture, the CLI updates the preset's frontmatter accordingly (a `--mark-signed` flag).
4. **Cluster-batch eligibility check** — `scripts/check-cluster-eligibility.ts` that verifies a cluster is mergeable (every preset's parity fixture is signed).

T-313 is the **last task in P13 Phase α**. After it merges, Phase α is complete and tracks β (preset library) and γ (frontier runtime) unlock.

## Architectural decisions

### D-T313-1. CLI: `generate-preset-parity-fixture.ts`

```
$ pnpm generate-parity-fixture --preset=cnn-classic [--mark-signed]
```

Flags:
- `--preset=<id>` — required; resolves via T-304's loader.
- `--mark-signed` — optional; on success, updates the preset's `signOff.parityFixture` field to `signed:YYYY-MM-DD`.
- `--frame=<n>` — optional; default canonical frame (e.g., 60 — mid-hold). Spec calls out single-frame v1.

Workflow:

1. Load preset via `loadPreset(skills/stageflip/presets/<cluster>/<preset>.md)`.
2. Build a `FixtureManifest` from the preset's `clipKind` + `preferredFont` + (resolved) `fallbackFont` from the font registry. Default composition: 1280x720, 30fps, 5s duration. Customizable via env vars in v2.
3. Invoke the existing parity-render pipeline (the same one `parity:prime` uses).
4. Output:
   - `parity-fixtures/<cluster>/<preset>/manifest.json` — the FixtureManifest.
   - `parity-fixtures/<cluster>/<preset>/golden-frame-<n>.png` — the rendered reference frame.
   - `parity-fixtures/<cluster>/<preset>/thresholds.json` — PSNR/SSIM thresholds (default values).
5. If `--mark-signed`, mutate the preset's MD frontmatter `signOff.parityFixture` to `signed:<today>`. (CLI prompts for confirmation before the write.)

### D-T313-2. Cluster eligibility check

```
$ pnpm check-cluster-eligibility --cluster=A
```

Walks the cluster, asserts every preset has `signOff.parityFixture: signed:YYYY-MM-DD`. Outputs a table:

```
Cluster A — News (8 presets)
  ✓ cnn-classic (signed:2026-04-29)
  ✓ bbc-reith-dark (signed:2026-04-29)
  ✗ fox-news-alert (pending-user-review)
  ...
Cluster A: NOT ELIGIBLE — 2 presets pending sign-off.
```

Exit 0 on eligible (all signed); exit 1 otherwise.

### D-T313-3. Sign-off workflow doc

`docs/ops/parity-fixture-signoff.md`:

1. **Generate** — `pnpm generate-parity-fixture --preset=<id>` produces artifacts in `parity-fixtures/<cluster>/<preset>/`.
2. **Inspect** — product owner views the golden frame, validates against the compass source for the preset.
3. **Sign** — `pnpm generate-parity-fixture --preset=<id> --mark-signed` updates frontmatter.
4. **Cluster batch merge** — `pnpm check-cluster-eligibility --cluster=<letter>` returns 0; cluster preset PRs may merge.

The doc is the canonical procedure; per ADR-004 §D5, ratification of the doc constitutes the product-owner sign-off authority being operationalized.

### D-T313-4. Frontmatter mutation safety

The `--mark-signed` flag mutates a markdown file. Guards:

- Read the file fresh before writing (no stale-cache races).
- Use the existing frontmatter parser (gray-matter via T-304 path) to ensure round-trip preserves body content + non-frontmatter formatting.
- Atomic write: write to a temp file in the same dir, then rename.
- Refuse to mutate if the current `signOff.parityFixture` is already `signed:...` UNLESS `--force` is supplied (re-sign requires explicit confirmation).

### D-T313-5. Determinism + browser-bundle posture

Script is Node-only. Reads `@stageflip/schema/presets/node` (T-304 subpath). The render side is exempt from determinism (the parity infra is the existing harness, not new code).

### D-T313-6. Output paths

- `parity-fixtures/<cluster>/<preset>/manifest.json`
- `parity-fixtures/<cluster>/<preset>/golden-frame-<n>.png`
- `parity-fixtures/<cluster>/<preset>/thresholds.json`

Top-level `parity-fixtures/` is committed (the goldens are part of the contract). `.gitignore` does NOT exclude this directory; consumers (parity tests in CI) read the goldens.

## Acceptance criteria

### `generate-preset-parity-fixture.ts`

1. `pnpm generate-parity-fixture --preset=cnn-classic`: resolves the preset via loader, builds manifest, invokes render pipeline, writes 3 artifacts (manifest.json, golden-frame-<n>.png, thresholds.json) at `parity-fixtures/news/cnn-classic/`.
2. **Unknown preset**: `--preset=mystery` exits non-zero with clear error.
3. **`--frame=<n>`**: overrides default canonical frame; output filename reflects.
4. **No render-pipeline access**: in test env without renderer-cdp, the CLI exits with a clear "rendering pipeline unavailable" message (the test path uses a stub renderer; the real pipeline runs operationally).

### `--mark-signed` flow (AC #5–#8)

5. **Mark signed**: `--mark-signed` updates the preset's `signOff.parityFixture` to `signed:YYYY-MM-DD` (today's date in UTC).
6. **Re-sign guard**: if already `signed:...`, exits non-zero unless `--force` supplied.
7. **Frontmatter preservation**: round-trip via gray-matter preserves body + ordering of unrelated frontmatter fields (verified by reading before/after diff).
8. **Atomic write**: a synthetic mid-write failure leaves the original file intact (best-effort — temp-file pattern).

### `check-cluster-eligibility.ts` (AC #9–#12)

9. `pnpm check-cluster-eligibility --cluster=A`: walks cluster A presets, returns table.
10. **All signed → eligible**: exit 0; output reports "Cluster A: ELIGIBLE".
11. **Some pending → not eligible**: exit 1; output lists pending presets.
12. **Unknown cluster**: `--cluster=Z` exits non-zero.

## Escalation triggers (CLAUDE.md §6)

- The render pipeline (`@stageflip/renderer-cdp`) requires Chrome+ffmpeg in test env. If the test env can't satisfy this, the CLI's render path must be fully stubbed and AC #4's "rendering pipeline unavailable" must be the test path. Verify before bypass.
- A real on-disk preset fails to produce a render (e.g., `staticFallback` references an unsupported clip kind). Surface as render error; don't silently produce an empty fixture.
- Atomic write for `--mark-signed` is infeasible cross-platform (Windows `rename` over an open file fails). Fall back to write-and-pray with a warning. Spec accepts this if documented.
- The fixture storage size grows past reason (50 presets × 1 PNG × N variants). Single-frame v1 is bounded; if multi-frame ships in v2, re-evaluate.
