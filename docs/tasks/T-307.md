# T-307 — Font-license registry

**Branch**: `task/T-307-font-license-registry`

## Goal

Stand up the **font-license registry** per ADR-004 §D3. T-307 ships:

1. **`packages/schema/src/presets/font-registry.ts`** — enumerates every font referenced across presets, its license, and (for `proprietary-byo` entries) the approved license-cleared fallback.
2. **License-vocabulary canonicalization** — promotes the deferred-from-T-304 license-string vocabulary (`commercial-byo`, `platform-byo`, `na`, `apache-2.0 + ofl`, etc.) into a typed enum + composite parser. Surfaces what `T-304 amendment v1` deferred.
3. **`check-licenses` extension** — verifies every preset's `fallbackFont.license` resolves to a whitelisted license per ADR-001 §D4.
4. **Registry entries** populated from on-disk preset stubs — a registry walk during build / CI extracts every `(preferredFont, fallbackFont)` pair from the 50+ stubs and produces a deduplicated registry.

T-307 builds on T-304's loader (`@stageflip/schema/presets/node`) — the registry is computed by walking the loaded preset registry, NOT by re-parsing files.

## Architectural decisions

### D-T307-1. License vocabulary — typed enum + composite parser

Per `feedback_t304_lessons.md` Cycle 1, T-304 deferred license vocabulary to `z.string().min(1)`. T-307 promotes this to typed:

```ts
export const fontLicenseAtomSchema = z.enum([
  'ofl',                  // SIL Open Font License
  'apache-2.0',
  'mit',
  'public-domain',
  'cc0-1.0',
  'proprietary-byo',      // bring-your-own (e.g., CNN Sans)
  'commercial-byo',       // licensed commercially; bring license
  'platform-byo',         // platform fonts (TikTok Sans, Apple SF) — assume installed
  'license-cleared',      // informal "we know it's OK" — escalate to type-design-consultant
  'license-mixed',        // multi-license fallback set
  'ofl-equivalent',       // OFL-compatible custom (e.g., custom 8-bit pixel fonts)
  'na',                   // text-free presets
]);

/**
 * License field in preset frontmatter. Either an atom or a composite expression.
 * Composite forms: 'ofl + proprietary-byo' (multi-license fallback group),
 * 'apache-2.0 / ofl / commercial-byo' (alternative-licensed alternatives).
 */
export const fontLicenseExpressionSchema = z.string().min(1);

export interface ParsedLicenseExpression {
  atoms: ReadonlyArray<z.infer<typeof fontLicenseAtomSchema>>;
  /** 'union' = any one of these is sufficient (e.g., a fallback-set where any atom works);
   *  'all' = all required (e.g., 'apache-2.0 + ofl' for a font that's dual-licensed). */
  composition: 'atom' | 'union' | 'all';
}

export function parseFontLicenseExpression(raw: string): ParsedLicenseExpression;
```

Parser handles:
- Single atom: `'ofl'` → `{ atoms: ['ofl'], composition: 'atom' }`.
- AND-composite (`+`): `'apache-2.0 + ofl'` → `{ atoms: ['apache-2.0', 'ofl'], composition: 'all' }`.
- OR-composite (`/`): `'apache-2.0 / ofl / commercial-byo'` → `{ atoms: ['apache-2.0', 'ofl', 'commercial-byo'], composition: 'union' }`.
- Mixed `+` and `/` reject — escalation trigger (no real-world stub uses both).
- Unknown atom: `'foo'` rejected with clear error.

### D-T307-2. Registry shape

```ts
export interface FontEntry {
  /** Canonical family name as it appears in preset frontmatter. */
  family: string;
  /** Parsed license expression. */
  license: ParsedLicenseExpression;
  /** When `license.atoms` includes 'proprietary-byo': the approved fallback to substitute at render. */
  approvedFallback?: { family: string; license: ParsedLicenseExpression; weight: number };
  /** Presets that reference this font (by frontmatter `id`). For audit. */
  referencedBy: string[];
}

export class FontLicenseRegistry {
  /** Indexed by canonical family name, lowercase, hyphen-spaced. */
  private byCanonicalName = new Map<string, FontEntry>();

  static buildFromPresets(presetRegistry: PresetRegistry): FontLicenseRegistry;
  get(family: string): FontEntry | undefined;
  list(): FontEntry[];
  /** Check if every preset's fonts resolve to whitelisted licenses. */
  validateAgainstWhitelist(allowedLicenses: ReadonlyArray<FontLicenseAtom>): { valid: boolean; violations: FontEntry[] };
}
```

Canonicalization rule: lowercase, replace whitespace + underscores with hyphens. `'CNN Sans'` → `'cnn-sans'`.

### D-T307-3. `check-licenses` extension

`scripts/check-licenses.ts` is amended to:

1. Load the preset registry (via `@stageflip/schema/presets/node`).
2. Build the font registry.
3. Validate against the ADR-001 §D4 whitelist + the font-license-specific atoms (`proprietary-byo`, `commercial-byo`, `platform-byo` permitted but flagged for `signOff.typeDesign`; `na` accepted; unknowns rejected).
4. Emit a violation list if any preset's font has an unwhitelisted license.

The script's existing function (validating npm dep licenses) is unchanged. T-307 ADDS a new validation block, not modifies the existing one.

### D-T307-4. Browser-bundle posture

The registry's `buildFromPresets(presetRegistry)` is **Node-only** because it consumes the loader (which is `@stageflip/schema/presets/node`). T-307 lives at `packages/schema/src/presets/font-registry.ts` BUT is exported via the `./presets/node` subpath, NOT the browser-safe main entry.

The license vocabulary types (atoms, expressions, parser) ARE browser-safe and can be exported from `@stageflip/schema` main entry.

Split:
- `packages/schema/src/presets/font-license.ts` — vocabulary + parser (browser-safe; main entry exports).
- `packages/schema/src/presets/font-registry.ts` — registry class + builder (Node-only; subpath export).

This mirrors T-304's split discipline.

### D-T307-5. Determinism posture

`packages/schema/**` is not in clip/runtime scope. The registry uses `Map`, `string` operations, no time/randomness. Deterministic by construction.

### D-T307-6. Test fixture handling

T-307's tests depend on the on-disk preset stubs. Two test paths:

- **Unit tests**: synthetic in-memory `PresetRegistry` — no on-disk dependency. Fast.
- **Integration tests**: `buildFromPresets(loadAllPresets('skills/stageflip/presets'))` — real-stub smoke. Like T-304 AC #37.

Integration test has its own AC: every existing preset stub's font references resolve in the registry without unknown-atom rejections. If a stub has an unknown atom, escalate (don't silently widen the enum — the enum is now the registry's contract).

## Acceptance criteria

### Vocabulary + parser (`font-license.ts`)

1. `fontLicenseAtomSchema` accepts all 12 atoms per D-T307-1; rejects unknown.
2. `parseFontLicenseExpression('ofl')`: returns `{ atoms: ['ofl'], composition: 'atom' }`.
3. `parseFontLicenseExpression('apache-2.0 + ofl')`: returns `{ atoms: ['apache-2.0', 'ofl'], composition: 'all' }`.
4. `parseFontLicenseExpression('apache-2.0 / ofl / commercial-byo')`: returns 3 atoms, composition `'union'`.
5. **Mixed `+` and `/`**: `'apache-2.0 + ofl / mit'` throws with a clear error message.
6. **Unknown atom**: `'foo'` throws.
7. **Whitespace tolerance**: `' ofl + apache-2.0 '` parses successfully.
8. **Empty string**: `''` throws.

### Registry (`font-registry.ts`)

9. `FontLicenseRegistry.buildFromPresets(presetRegistry)`: walks every preset, extracts `preferredFont` + `fallbackFont`, deduplicates by canonical family name.
10. **Canonicalization**: `'CNN Sans'`, `'cnn sans'`, `'cnn-sans'` all collapse to `'cnn-sans'` in the registry.
11. **`approvedFallback` populated**: when a preset has `preferredFont.license` containing `proprietary-byo` AND `fallbackFont` is set, the fallback is attached to the `preferredFont`'s registry entry.
12. **`referencedBy` aggregation**: every preset id that references a given font appears in the entry's `referencedBy`.
13. `get(family)` is canonicalization-aware: `get('CNN Sans')` and `get('cnn-sans')` return the same entry.
14. `list()` returns sorted entries (deterministic).
15. **`validateAgainstWhitelist(allowed)`**: returns valid + violations. Pin via fixture with mixed-licence presets.

### `check-licenses` extension

16. `scripts/check-licenses.ts` (extended) PASS at HEAD: every preset's `fallbackFont.license` resolves to a whitelisted atom OR is a documented `proprietary-byo` paired with an approved fallback.
17. **Violation surfacing**: an injected synthetic preset with an unknown license atom causes the script to exit non-zero with the violation listed. Pin via test (the script must remain fast — gate-suitable).

### Integration test (real stubs)

18. **Smoke**: `buildFromPresets(loadAllPresets('skills/stageflip/presets'))` against the actual on-disk presets succeeds. NO unknown atoms surface (per Cycle 1 lesson — the enum was authored from the on-disk vocabulary).

20. Every `proprietary-byo` font in the on-disk registry has a populated `approvedFallback` OR is flagged as missing-fallback (audit).

## Escalation triggers (CLAUDE.md §6)

- A real on-disk preset stub uses a license atom not in the 12-element enum. Per `feedback_t304_lessons.md` Cycle 1+2, this should NOT happen — the enum was authored from observed vocabulary. If it does, escalate; either widen enum (with rationale) or fix the stub.
- A preset uses BOTH `+` and `/` composite operators in one expression. Escalate; v1 doesn't support; the spec rejects this.
- A `proprietary-byo` font has no `fallbackFont` in its preset. Per ADR-004 §D3 + AC #20, surface as audit-flag (NOT hard error). T-308's gate may upgrade to hard error.
- `check-licenses` runtime exceeds CI budget (>30s). Optimize the registry build or memoize. Escalate if structural.
