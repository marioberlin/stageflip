# T-388 — `VoiceClip` `staticFallback` (waveform poster)

**Branch**: `task/T-388-voice-clip-static-fallback`

## Goal

Ship the `staticFallback` rendering for `VoiceClip` (T-387's sister task). When the permission gate denies `mic` or the tenant policy refuses, the mount-harness routes to a deterministic poster: a stylised waveform silhouette + a "tap-to-speak" overlay text slot. Consumed by the renderer-cdp / video / slide export paths where live mic capture is impossible.

T-388 is the `staticFallback` half of ADR-005 §D1's `VoiceClip` row. T-387 ships the `liveMount`; T-388 closes the pair so `family: 'voice'` is structurally complete.

## Architectural decisions

### D-T388-1. Schema addition — `posterText?` slot on `voiceClipPropsSchema`

T-387 shipped `voiceClipPropsSchema` with `posterFrame`. T-388 adds an optional `posterText` overlay copy field:

```ts
// packages/schema/src/clips/interactive/voice-props.ts (MODIFIED)
export const voiceClipPropsSchema = z.object({
  // ... existing T-387 fields ...
  /** Overlay copy rendered above the waveform poster on the static path. App-supplied; no English defaults. */
  posterText: z.string().min(1).optional(),
}).strict();
```

Browser-safe. Existing T-387 props consumers continue to validate (the field is optional).

### D-T388-2. Default `staticFallback` shape — `ImageElement` + `TextElement`

When `voiceClip.staticFallback` is empty (no Implementer authoring), the mount-harness routing path is already responsible for selecting a fallback. T-388's contribution is a **default-poster generator** that produces the canonical `Element[]` shape for `family: 'voice'` clips when an author does not supply their own:

```ts
// packages/runtimes/interactive/src/clips/voice/static-fallback.ts (NEW)
export function defaultVoiceStaticFallback(args: {
  width: number;
  height: number;
  posterText?: string;
  /** PRNG seed used to deterministically generate the silhouette geometry. Default 0. */
  silhouetteSeed?: number;
}): Element[];
```

Returns:
1. An `ImageElement` whose `src` is a deterministic data URL of an SVG waveform silhouette generated from the seeded PRNG.
2. A `TextElement` rendering `posterText` (when present) centred over the image.

The function is **deterministic** — same `(width, height, posterText, silhouetteSeed)` produces the same Element tree byte-for-byte. Subject to the broad §3 rule (no random / no time).

### D-T388-3. SVG silhouette — pure-deterministic generator

Authors-supplied posters are out of scope (cluster-author tooling). T-388's default generates a minimal SVG silhouette:

- Grid of evenly-spaced bars (e.g., 32 bars across the width).
- Each bar's height is `clamp(prng(i), 0.1, 1.0) * height * 0.4` centred vertically.
- Output is a `data:image/svg+xml,...` URL.
- The SVG markup is generated synchronously via string concatenation; no DOM, no canvas. Browser-safe and Node-safe.

Reuse the seeded PRNG primitive shipped by T-384 (`createSeededPRNG` from `runtimes-interactive/src/clips/three-scene/prng.ts`) — extract it to a shared location if T-388's adoption makes it the third application of the PRNG primitive (per CLAUDE.md "three similar lines beat a premature abstraction"). The third use **is** T-388 (T-384 + T-388 + planned future use), so extraction earns its place — move PRNG to `packages/runtimes/interactive/src/prng.ts` as a peer of `frame-source.ts`.

### D-T388-4. No new factory; integration via mount-harness fallback path

T-388 does NOT register a new clip factory. The `voice` family factory (T-387) detects the `staticFallback` routing decision from the harness and delegates to `defaultVoiceStaticFallback` when the clip's authored `staticFallback` is empty. The mount-harness's existing fallback rendering path consumes the returned `Element[]`.

### D-T388-5. Telemetry

- `voice-clip.static-fallback.rendered` (attributes: `reason: 'permission-denied' | 'tenant-denied' | 'authored'`, `width`, `height`, `posterTextLength` (NOT the text body — privacy))

## Acceptance criteria

### Schema

1. `voiceClipPropsSchema` accepts `posterText` of any non-empty string.
2. `posterText: ''` → schema error.
3. Existing T-387 fixtures (without `posterText`) continue to validate.

### Generator determinism

4. `defaultVoiceStaticFallback({ width: 640, height: 360 })` returns the same Element[] byte-for-byte across two calls.
5. Different `silhouetteSeed` → different SVG markup.
6. The returned ImageElement's `src` is a `data:image/svg+xml,…` URL.
7. The returned TextElement's text matches the supplied `posterText`; absent `posterText` → no TextElement in the array.

### Bounds

8. The image element's transform width/height match the supplied dimensions.
9. The text element is centred (transform.x + width/2 within ±1 px of the image centre).

### PRNG extraction (D-T388-3)

10. `createSeededPRNG` exported from `packages/runtimes/interactive/src/prng.ts` produces identical sequences to the pre-extraction location for the same seed (regression pin).
11. The legacy import path `from './prng.js'` inside `clips/three-scene/` continues to resolve via the re-export (T-384 backward-compat).

### Factory integration

12. `voice` factory mounts with `staticFallback: []` on schema → harness routes to `defaultVoiceStaticFallback` → renders the default Element[].
13. `voice` factory mounts with `staticFallback: [<authored Element>]` → harness uses the authored value (default generator NOT called).

### Telemetry

14. `voice-clip.static-fallback.rendered` emits with documented attribute shape; `posterTextLength` is the integer length, not the text body.

### Determinism gate

15. `pnpm check-determinism` PASS — `static-fallback.ts` lives at `clips/voice/**` (not `clips/{shader,three-scene}/**`); shader sub-rule does not apply but broad §3 rule does.
