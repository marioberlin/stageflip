# T-305 — Interactive-clip contract

**Branch**: `task/T-305-interactive-clip-contract`

## Goal

Stand up the **schema layer** for the interactive runtime tier per ADR-003 §D2/D3/D4. T-305 ships:

1. **`InteractiveClip` schema** at `packages/schema/src/clips/interactive.ts` — Zod types for the discriminated `staticFallback + liveMount + permissions` shape.
2. **`ExportTarget` enum** at `packages/schema/src/clips/export-targets.ts` — the closed enum of export targets (MP4, PPTX, HTML, display-pre-rendered, display-interactive, on-device-player) per ADR-003 §D3.
3. **`resolveClipPath(target, clip)`** export-matrix function — returns `'static' | 'live'` for routing the export pipeline.
4. **`Permission` enum** for the permission envelope per ADR-003 §D4: `'mic' | 'network' | 'camera'` (open to extension via ADR amendment).
5. **`componentRefSchema`** — typed reference to a runtime component the interactive tier can mount.

T-305 does NOT ship the runtime tier package (`packages/runtimes/interactive/` is T-306). T-305 is the **schema/contract layer** the runtime + the export matrix consume.

## Architectural decisions

### D-T305-1. `InteractiveClip` is a discriminated extension of `ClipElement`

Per ADR-003 §D2 + ADR-005 §D2, frontier clips declare BOTH paths. Schema:

```ts
export const permissionSchema = z.enum(['mic', 'network', 'camera']);
export type Permission = z.infer<typeof permissionSchema>;

export const componentRefSchema = z.object({
  /** Package-qualified component identifier — e.g. `@stageflip/runtimes-interactive/clips/shader#ShaderClip`. */
  module: z.string().regex(/^@?[a-z0-9-]+(?:\/[a-z0-9-]+)*#[A-Z][A-Za-z0-9_]*$/),
  /** Optional version pin (semver range) for deployment-time validation. */
  version: z.string().optional(),
}).strict();

export const interactiveClipSchema = z.object({
  type: z.literal('interactive-clip'),
  /** Frontier clip family — narrows further than just "interactive". */
  family: z.enum(['shader', 'three-scene', 'voice', 'ai-chat', 'live-data', 'web-embed', 'ai-generative']),
  staticFallback: z.array(canonicalElementSchema).min(1, 'interactive clip must declare a non-empty staticFallback'),
  liveMount: z.object({
    component: componentRefSchema,
    props: z.record(z.unknown()),  // typed by family-specific extensions in Phase γ
    permissions: z.array(permissionSchema).default([]),
  }),
  posterFrame: z.number().int().nonnegative().optional(),
}).strict();
```

The `staticFallback` non-empty refine implements ADR-003 §D2 invariant ("declared only `liveMount` is rejected") at the type level. `check-preset-integrity` (T-308) enforces this at CI level too.

`liveMount.props` is `z.record(z.unknown())` in T-305. Phase γ tasks (per-family clip implementations) extend with discriminated unions keyed by `family`.

### D-T305-2. Export matrix per ADR-003 §D3

```ts
export const exportTargetSchema = z.enum([
  'mp4',
  'image-sequence',
  'pptx-flat',
  'html-slides',
  'live-presentation',
  'display-pre-rendered',
  'display-interactive',
  'on-device-player',
]);
export type ExportTarget = z.infer<typeof exportTargetSchema>;

export type ResolvedClipPath = 'static' | 'live';

const EXPORT_MATRIX: Readonly<Record<ExportTarget, ResolvedClipPath>> = {
  'mp4':                  'static',
  'image-sequence':       'static',
  'pptx-flat':            'static',
  'html-slides':          'live',
  'live-presentation':    'live',
  'display-pre-rendered': 'static',
  'display-interactive':  'live',
  'on-device-player':     'live',
};

export function resolveClipPath(target: ExportTarget, clip: InteractiveClip): ResolvedClipPath {
  return EXPORT_MATRIX[target];
}
```

The function takes the clip parameter for forward-compat (a future per-clip override could short-circuit), but the v1 implementation is target-only.

### D-T305-3. Browser-safe surface

Per `feedback_t304_lessons.md`: T-305 adds schemas to `packages/schema/`. The package is bundled for browser apps. **No `fs` / `path` / `child_process` imports.** Pure Zod.

The contract is intentionally schema-only, no I/O. The runtime tier (T-306) handles the I/O side.

### D-T305-4. Determinism posture

The schema is browser-safe and the resolver is pure. `check-determinism` doesn't scan `packages/schema/**` (it's not in the clip/runtime subdirectories). No determinism implications.

ADR-003 §D5 says `check-determinism` will gain a shader sub-rule applying inside the interactive tier (T-306-era enforcement). T-305 is the type-level prerequisite; the runtime sub-rule lands in T-309.

### D-T305-5. Existing `ClipElement` integration

`packages/schema/src/elements/index.ts` has a `ClipElement` union (existing — matches `BlenderClip`, etc.). T-305 adds `InteractiveClip` to this union as another discriminated branch. Keep the union strict so unknown `type` values throw at parse.

The union update is non-trivial; T-305 verifies that the existing element-roundtrip tests still pass after the addition.

## Acceptance criteria

Each gets a Vitest test, written first and failing.

### `interactive.ts`

1. `interactiveClipSchema.parse({ ...valid })` accepts a complete InteractiveClip per D-T305-1.
2. **Empty staticFallback**: parsing with `staticFallback: []` throws with the spec's `min(1)` message.
3. **Missing liveMount**: omitting `liveMount` throws.
4. **Unknown family**: `family: 'unknown'` throws (closed enum).
5. **componentRefSchema** strict format: `'@scope/pkg/sub#ClassName'` or `'pkg#ClassName'` accepts; `'just-a-string'` or `'pkg#lowercase'` (component must be PascalCase) throws.
6. **Permissions default to `[]`** when omitted.
7. **All three permissions** (`mic`, `network`, `camera`) accepted; unknown permission rejected.
8. **`posterFrame` optional**: omission OK; negative value throws.
9. **`liveMount.props`** accepts arbitrary record (no schema enforcement at this layer).

### `export-targets.ts`

10. `exportTargetSchema.parse('mp4')` accepts; `'unknown'` throws.
11. `EXPORT_MATRIX` has an entry for every member of `exportTargetSchema.options`. Pin via test that iterates.
12. `resolveClipPath('mp4', anyClip)` returns `'static'`.
13. `resolveClipPath('html-slides', anyClip)` returns `'live'`.
14. **All 8 targets** match ADR-003 §D3 routing exactly. Pin a test table.

### Element union integration

15. `elementSchema.parse({ ...interactiveClip })` accepts InteractiveClip via the discriminated union (the `type: 'interactive-clip'` branch).
16. Existing element fixtures (text, image, shape, group, table, blender-clip, etc.) still parse unchanged. Pin via existing roundtrip tests passing.
17. **Unknown clip type**: `type: 'mystery-clip'` throws (the union is strict).

## Escalation triggers (CLAUDE.md §6)

- Adding `InteractiveClip` to the element union breaks an existing roundtrip test in a way that's not a missing-discriminator issue. Escalate; the union shape may need refactoring.
- The 7 frontier families enum doesn't match what ADR-005 §D1 enumerates. Escalate; the family enum is part of the spec contract — don't drift silently.
- A future per-target override on `resolveClipPath` is needed BEFORE T-305 merges (e.g., a specific clip family can't run on `display-interactive`). Escalate; v1 is target-only.
- The `componentRefSchema` regex (`'@?[a-z0-9-]+(?:/[a-z0-9-]+)*#[A-Z][A-Za-z0-9_]*'`) doesn't accommodate a real component path that ADR-005 implies. Escalate; the regex is part of the schema contract.
