# T-388a — Static-fallback generator registry

**Branch**: `task/T-388a-static-fallback-registry`

## Goal

Replace the family-hardcoded `if (clip.family !== 'voice')` branch in `mount-harness.ts:resolveStaticFallbackElements` (T-388 / PR #280) with a **per-family registry** parallel to `interactiveClipRegistry`. Each γ-live family that supplies a default-poster generator registers it at module-load time (side-effect, same pattern as the clip factory); the harness looks up and dispatches generically.

T-388a must land before T-389 (`AiChatClip`) starts so the next γ-live family registers a generator instead of accumulating another `if`-branch in the harness. Same workflow shape as T-309a's relationship to T-383: ship the immediate work with a known workaround, fix the workaround before the pattern propagates.

## Architectural decisions

### D-T388a-1. New `staticFallbackGeneratorRegistry` parallel to `interactiveClipRegistry`

```ts
// packages/runtimes/interactive/src/static-fallback-registry.ts (NEW)
import type { Element, InteractiveClip, InteractiveClipFamily } from '@stageflip/schema';

export interface StaticFallbackGeneratorContext {
  clip: InteractiveClip;
  /** Routing reason (`'permission-denied' | 'tenant-denied' | 'pre-prompt-cancelled' | 'authored'`) — generators may use this for telemetry. */
  reason: string;
  /** Telemetry sink — same `MountContext.emitTelemetry` shape. */
  emitTelemetry: (event: string, attributes: Record<string, unknown>) => void;
}

/** Returns the default Element[] when the clip's authored `staticFallback` is empty. */
export type StaticFallbackGenerator = (ctx: StaticFallbackGeneratorContext) => ReadonlyArray<Element>;

export class StaticFallbackGeneratorRegistry {
  private readonly generators = new Map<InteractiveClipFamily, StaticFallbackGenerator>();
  register(family: InteractiveClipFamily, generator: StaticFallbackGenerator): void { /* throws on dup */ }
  resolve(family: InteractiveClipFamily): StaticFallbackGenerator | undefined { /* lookup */ }
  list(): InteractiveClipFamily[] { /* sorted */ }
  unregister(family: InteractiveClipFamily): boolean { /* test seam */ }
  clear(): void { /* test seam */ }
}

export const staticFallbackGeneratorRegistry = new StaticFallbackGeneratorRegistry();
```

Module-level singleton. Same lifecycle as `interactiveClipRegistry`. Throws `StaticFallbackGeneratorAlreadyRegisteredError` on duplicate registration; permits `unregister`/`clear` only as test seams.

### D-T388a-2. Voice family registers its generator via the registry

`packages/runtimes/interactive/src/clips/voice/index.ts` gains a side-effect call:

```ts
import { staticFallbackGeneratorRegistry } from '../../static-fallback-registry.js';
import { defaultVoiceStaticFallback } from './static-fallback.js';

// existing factory registration ...
interactiveClipRegistry.register('voice', voiceClipFactory);

// NEW
staticFallbackGeneratorRegistry.register('voice', ({ clip, reason, emitTelemetry }) => {
  const props = (clip.liveMount.props ?? {}) as { posterText?: unknown };
  const posterText = typeof props.posterText === 'string' ? props.posterText : undefined;
  const generated = defaultVoiceStaticFallback({
    width: clip.transform.width,
    height: clip.transform.height,
    ...(posterText !== undefined ? { posterText } : {}),
  });
  emitTelemetry('voice-clip.static-fallback.rendered', {
    family: 'voice',
    reason,
    width: clip.transform.width,
    height: clip.transform.height,
    posterTextLength: posterText !== undefined ? posterText.length : 0,
  });
  return generated;
});
```

The `defaultVoiceStaticFallback` core function stays unchanged; only the wiring moves.

### D-T388a-3. Harness becomes family-agnostic

`mount-harness.ts:resolveStaticFallbackElements` replaces the hardcoded branch:

```ts
private resolveStaticFallbackElements(clip: InteractiveClip, reason: string): ReadonlyArray<Element> {
  if (clip.staticFallback.length > 0) {
    // Authored fallback wins. If the family has a generator registered,
    // call it with reason='authored' so the family's telemetry still
    // fires on the authored path; otherwise pass through unchanged.
    const generator = staticFallbackGeneratorRegistry.resolve(clip.family);
    if (generator !== undefined) {
      generator({
        clip,
        reason: 'authored',
        emitTelemetry: this.emitTelemetry,
      });
    }
    return clip.staticFallback;
  }
  const generator = staticFallbackGeneratorRegistry.resolve(clip.family);
  if (generator === undefined) {
    // No generator registered AND no authored fallback — fall through to
    // the schema's non-empty refine, which would have rejected this at
    // parse time. Defensive: return the empty array; the harness's React
    // root renders nothing.
    return clip.staticFallback;
  }
  return generator({
    clip,
    reason,
    emitTelemetry: this.emitTelemetry,
  });
}
```

No more `if (clip.family !== 'voice')`. T-389/T-391/T-393/T-395 register their own generators (or don't, if they don't ship a default) without touching the harness.

## Acceptance criteria

### Registry mechanics

1. `staticFallbackGeneratorRegistry.register('voice', gen)` succeeds.
2. Re-registering the same family throws `StaticFallbackGeneratorAlreadyRegisteredError`.
3. `resolve('voice')` returns the registered generator after the side-effect import.
4. `resolve('ai-chat')` returns `undefined` when no generator is registered.
5. `list()` returns sorted family names.
6. `unregister('voice')` returns `true`; subsequent `resolve` returns `undefined`.
7. `clear()` removes all registrations (test seam).

### Harness integration

8. With voice generator registered, mounting a clip with `family: 'voice'` and empty `staticFallback` calls the registered generator with `reason` matching the routing decision.
9. With voice generator registered, mounting a clip with `family: 'voice'` and authored `staticFallback` calls the generator with `reason: 'authored'` (same shape T-388 had) AND uses the authored array for rendering.
10. Mounting a clip with `family: 'ai-chat'` (no generator registered) and empty `staticFallback` returns the (empty) authored array; no error thrown.
11. The `if (clip.family !== 'voice')` literal no longer appears in `mount-harness.ts` (grep assertion in test).

## Escalation triggers (CLAUDE.md §6)

- The registry singleton creates a test-isolation problem (cross-test state pollution because module-load registers once globally). Escalate; the test approach may need an explicit `clear()` in `beforeEach`.
- A future γ-live family has TWO sensible default generators (e.g., live-data with both a "loading" and "stale-cached" fallback) and the single-generator-per-family contract doesn't fit. Escalate; we may need a generator-with-variant-key model.
- The harness's `emitTelemetry` is a method bound to `this`; passing it to the generator without `.bind(this)` breaks. Escalate; design the Context interface accordingly.
