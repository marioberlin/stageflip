# T-246 — AI-QC convergence loop (classical CV first, Gemini second)

**Branch**: `task/T-246-ai-qc-convergence-loop`

## Goal

Convert `@stageflip/import-google-slides`'s `tree.pendingResolution` array (populated by T-244 when deterministic matching fell below `matchConfidenceThreshold`) into resolved canonical element values, by sending one targeted multimodal Gemini call per residual.

Per plan v1.24 row at `docs/implementation-plan.md:534`: **classical CV first, Gemini second**. ~90% of objects are expected to resolve in the cheap deterministic pass shipped in T-244 (PaddleOCR text-content match, OpenCV / SAM 2 mask containment, z-order plausibility). Only the residual ~10% reach T-246's Gemini loop. This bounds Gemini token cost to a small per-element prompt rather than a per-slide or per-deck dump, and yields a meaningful per-element confidence on every imported deck.

The convergence loop iterates: each pending residual gets one Gemini call (cropped PNG + API metadata + structured question); if the model returns values that satisfy the per-element confidence threshold, the resolved values are written back into the `Document` and `pendingResolution` clears the entry. If the model fails (low confidence, malformed response, API error), the residual stays + a `LF-GSLIDES-LOW-MATCH-CONFIDENCE` flag is emitted at the end.

T-246 ships as a **post-walk pass** on `@stageflip/import-google-slides` (mirrors T-243a's `parsePptx → resolveAssets` split). Callers compose:

```ts
const tree = await parseGoogleSlides({ presentationId, auth, cv });
const aiqcTree = await runAiQcConvergence(tree, { llm: geminiProvider });
const resolved = await resolveAssets(aiqcTree, fetcher, firebaseStorage);
```

Each pass is independent; tests replace `llm` with a stub provider returning canned responses.

## Out of scope

| Item | Why deferred |
|---|---|
| Multi-call refinement per residual | v1 sends one Gemini call per residual. If the model's first response misses the confidence threshold, T-246 emits `LF-GSLIDES-LOW-MATCH-CONFIDENCE` and accepts the residual unresolved. Multi-step reasoning loops (the model asks a clarifying question, etc.) are out of scope. |
| Cross-element batching | Each residual is sent to Gemini separately. Batching ~10 residuals into one prompt would cut latency but balloon the prompt token count and confuse the model on which crop corresponds to which element. v1 = one call per residual. |
| Deck-level questions ("what's the overall theme?") | The convergence loop is per-element; whole-deck questions are a separate v2 surface (e.g., a `runAiDeckQc(doc, ...)` task). |
| Anthropic / OpenAI fallback for the convergence pass | T-246 talks to Gemini specifically (the plan-row commits to "Gemini multimodal"). The provider seam in `@stageflip/llm-abstraction` would let a future task swap in Claude Vision or GPT-4o; not in v1. |
| Gemini token-cost telemetry | Cost tracking is real but lives at the `LLMProvider` seam (each provider already carries `usage.input_tokens` / `output_tokens` per response). T-246 does NOT emit telemetry events; consumers can sum `usage` themselves. |
| Streaming responses | The Gemini call is `request → response`, not streamed. Streaming would complicate the JSON-shape validation T-246 does on the response. v1 = batch only. |
| Image-fallback rasterization on Gemini failure | If Gemini fails, T-246 emits `LF-GSLIDES-LOW-MATCH-CONFIDENCE` and leaves the element with API-only values. T-245's `rasterizeFromThumbnail` is NOT invoked here; T-246 doesn't have the page PNG (only the per-element crop). The image-fallback path runs at a different layer (T-244's parser emits `LF-GSLIDES-IMAGE-FALLBACK` for elements that can't be modeled cleanly; T-246 only handles low-match-confidence residuals). |

## Architectural decisions

### 1. Multimodal-image extension to `@stageflip/llm-abstraction` — bundled prereq

The current `LLMContentBlock` union (`packages/llm-abstraction/src/types.ts:8-15`) is `text | tool_use | tool_result` — no image support. T-246 needs to send a cropped PNG to Gemini.

**Decision**: bundle a small additive extension to the abstraction in this PR rather than splitting into a sub-task. The extension:

```ts
// packages/llm-abstraction/src/types.ts (added)
export type LLMContentBlock =
  | { type: 'text'; text: string }
  | { type: 'image'; mediaType: 'image/png' | 'image/jpeg' | 'image/webp'; data: string }  // NEW: data is base64
  | { type: 'tool_use'; id: string; name: string; input: unknown }
  | { type: 'tool_result'; tool_use_id: string; content: string; is_error?: boolean };
```

The Google provider (`packages/llm-abstraction/src/providers/google.ts`) translates the `image` block to Gemini's `inlineData: { mimeType, data }` shape; Anthropic + OpenAI providers translate to their respective image-input shapes. **T-246 only ships the Google-side binding**; Anthropic + OpenAI image bindings are a follow-on task — emit `LLMError({kind: 'unsupported'})` for those providers when given an image block.

**Backward compat**: existing callers that don't pass image blocks see no behavior change. The provider tests already cover text-only requests; the new `image` branch gets its own tests in T-246's PR.

**Why bundle in T-246**: alternative is a separate `T-246-multimodal-image` micro-task. The extension is ~50 LoC across two files; splitting forces a 2-PR sequence with no benefit. The bundled extension is the smaller path.

### 2. Public surface

```ts
// packages/import-google-slides/src/aiqc/index.ts

export interface RunAiQcConvergenceOptions {
  /** LLM provider configured for Gemini. Required. */
  llm: LLMProvider;
  /** Gemini model id. Default: 'gemini-2.0-flash' (cost-efficient multimodal). */
  model?: string;
  /**
   * Confidence threshold for accepting a Gemini-resolved value. Default 0.85
   * (higher than T-244's 0.78 deterministic threshold — Gemini needs to be
   * MORE confident than the deterministic matcher was, since this is the
   * fallback).
   */
  acceptThreshold?: number;
  /** Max tokens per Gemini call. Default 1024. */
  maxTokens?: number;
  /** Per-call timeout in ms. Default 30000. */
  timeoutMs?: number;
  /** Cost cap: max Gemini calls across the whole deck. Excess residuals get LF-GSLIDES-LOW-MATCH-CONFIDENCE without being attempted. Default 100. */
  maxCallsPerDeck?: number;
}

export interface RunAiQcConvergenceResult {
  /** Tree with resolved values written back into elements. `pendingResolution` is updated to drop entries that resolved cleanly. */
  tree: CanonicalSlideTree;
  /** Loss flags emitted during the convergence pass. */
  lossFlags: LossFlag[];
  /** Number of Gemini calls actually made (may be < pendingResolution.length if maxCallsPerDeck capped). */
  callsMade: number;
  /** Per-element resolution result for auditing / metrics. */
  resolutions: AiQcResolution[];
}

export interface AiQcResolution {
  slideId: string;
  elementId: string;
  /** 'resolved' | 'rejected-low-confidence' | 'rejected-llm-error' | 'skipped-cap'. */
  outcome: 'resolved' | 'rejected-low-confidence' | 'rejected-llm-error' | 'skipped-cap';
  /** Gemini's reported confidence. Undefined when outcome is 'skipped-cap' or 'rejected-llm-error'. */
  geminiConfidence?: number;
  /** Underlying error code when outcome is 'rejected-llm-error'. */
  errorCode?: 'API_ERROR' | 'MALFORMED_RESPONSE' | 'TIMEOUT';
}

export function runAiQcConvergence(
  tree: CanonicalSlideTree,
  opts: RunAiQcConvergenceOptions,
): Promise<RunAiQcConvergenceResult>;
```

The function is **pure** relative to its inputs given a deterministic `LLMProvider` (the test stub returns canned responses). Real Gemini calls are inherently non-deterministic; production tests use the stub.

### 3. Per-element prompt structure

For each `PendingMatchResolution` entry, T-246 builds a Gemini multimodal request:

```
System prompt (constant):
  "You are a slide-deck import assistant. You receive a cropped image of one
  element from a Google Slides page, plus the API metadata for that element.
  Your job is to identify what the element represents (text content,
  shape kind, fill color, etc.) and return a structured JSON response
  matching the provided schema. Be conservative: if you are not sure,
  return confidence < 0.85 and explain why."

User prompt (per residual):
  - One image block (the pageImageCropPx slice, base64-encoded PNG)
  - One text block:
      "API metadata for this element:
       <JSON dump of apiElement, formatted>

       The deterministic matcher's top candidate was:
       <JSON dump of rankedCandidates[0]>

       Question: provide a JSON response with the resolved element values.
       Schema:
       {
         confidence: number 0..1,
         resolvedKind: 'text' | 'shape' | 'image' | 'group' | 'table' | 'other',
         text: string | null,
         fillColor: string | null,  // hex like '#FF0000', or null if no clear fill
         shapeKind: string | null,  // e.g. 'rect', 'rounded-rect', 'ellipse', or null
         reasoning: string  // brief, 1-2 sentences
       }
       Return ONLY the JSON object, no surrounding markdown."
```

The response is parsed with a Zod validator at the boundary; malformed JSON → `outcome: 'rejected-llm-error'` with `errorCode: 'MALFORMED_RESPONSE'`.

### 4. Resolution writeback

When Gemini returns `confidence >= acceptThreshold`:

- **`resolvedKind === 'text'`** + non-null `text`: update the element's `text` field; if the API element was emitted as a `ShapeElement` but Gemini says it's text, **convert the element type** in the tree (rare; pin via fixture).
- **`resolvedKind === 'shape'`** + non-null `shapeKind`: update the element's `shape` field (canonical names: `'rect' | 'rounded-rect' | 'ellipse' | 'line' | 'arrow'`; map `rounded-rect` → `'roundRect'` per schema).
- **Non-null `fillColor`**: update the element's fill color via the schema's color field (depends on element type — `ShapeElement.fill`, etc.).
- The element's deterministic-confidence value is replaced with `geminiConfidence`; `inheritsFrom` is preserved unchanged.
- The corresponding `pendingResolution` entry is removed from the tree.

When Gemini returns `confidence < acceptThreshold` OR malformed response OR API error:

- The element keeps its API-only values (unchanged from T-244's emission).
- The corresponding `pendingResolution` entry stays.
- `LF-GSLIDES-LOW-MATCH-CONFIDENCE` is emitted (one per unresolved residual); the existing flag from T-244 is reused — no new code introduced.

### 5. Cost cap + ordering

`maxCallsPerDeck` caps total Gemini calls. When a deck has more residuals than the cap allows:

- Residuals are processed in slide-order, then within-slide element-order.
- Once `callsMade === maxCallsPerDeck`, remaining residuals are skipped with `outcome: 'skipped-cap'` and emit `LF-GSLIDES-LOW-MATCH-CONFIDENCE`.
- A summary `LF-GSLIDES-AI-QC-CAP-HIT` (new code, severity `warn`) emits once at the deck level when any residual was skipped.

Default cap of 100 covers typical decks; larger decks can override.

### 6. Convergence semantics — **single-pass**, not iterative

The plan-row v1.24 says "Loop iterates until every element has a confidence above threshold or `LF-GSLIDES-LOW-MATCH-CONFIDENCE` is emitted for the residual." On re-reading, the "loop" terminology is from a higher-level abstract view; the concrete implementation is a **single pass** through `pendingResolution`:

- Each residual gets exactly one Gemini call.
- If the call resolves cleanly, the entry is removed.
- If the call fails or is low-confidence, the entry stays + the loss flag is emitted.
- No retries on the same element across iterations (a retry would just produce the same tokens at the same temperature).

A real "iterative" loop would re-deterministic-match after AI fills in some elements (in case the now-known-text on element A helps deterministic match element B). That second-order effect is real but rare; v1 ships single-pass.

### 7. Loss flags — one new code; reuse existing

Reuse from T-244 (`source: 'gslides'`):

- `LF-GSLIDES-LOW-MATCH-CONFIDENCE`: emitted once per unresolved residual after T-246 ran (rejected, malformed, capped, or low-confidence Gemini response).

New code in this PR:

| Code | Severity | Category | Emitted when |
|---|---|---|---|
| `LF-GSLIDES-AI-QC-CAP-HIT` | `warn` | `other` | The deck had more residuals than `maxCallsPerDeck`; one summary flag at the deck level. |

The wrapper `emitLossFlag` already auto-fills `source: 'gslides'` from T-244; T-246 just adds the new code to the `CODE_DEFAULTS` table.

### 8. Tests-first contract — stub LLMProvider

The test surface centers on a `StubGeminiProvider` (`packages/import-google-slides/src/aiqc/stub-provider.ts`) that returns canned `LLMResponse` payloads keyed by request hash. Each fixture pins:

- A `tree` with non-empty `pendingResolution`.
- A canned response per residual (`{ confidence, resolvedKind, text, ... }` JSON encoded as the model's text block).
- The expected `RunAiQcConvergenceResult`.

No live Gemini calls in CI. The Google provider's image-block translation is tested separately with a `GeminiClientLike` mock that asserts the `inlineData` shape.

### 9. Determinism

The pass is deterministic given a deterministic `LLMProvider`. Sources of non-determinism (controlled):

- The Gemini API itself is non-deterministic. Production tests use the stub.
- Response-parsing: Zod validation is deterministic.
- Iteration order over `pendingResolution`: sorted by `slideId` then `elementId` to ensure stable ordering across runs.

This package is not in CLAUDE.md §3's determinism-restricted scope; the source-level grep test (mirrors T-253-base AC #28 + T-245 AC #19 + T-247 AC #33) enforces no `Date.now` / `Math.random` / `performance.now` etc. in `packages/import-google-slides/src/aiqc/**`.

9. Per-element prompt's user message contains exactly one `image` block + exactly one `text` block (in that order). Pin via fixture.

    - The API metadata as JSON (`apiElement` field of `PendingMatchResolution`)
    - The top deterministic candidate as JSON (`rankedCandidates[0]` field)

    Pin via substring assertions on the expected fields.

### Response validation

12. **Valid response** (matches Zod schema, `confidence` in `[0, 1]`, `resolvedKind` in the union): parses cleanly. Pin.
13. **Confidence out of range** (`confidence: 1.5`): rejected with `outcome: 'rejected-llm-error'`, `errorCode: 'MALFORMED_RESPONSE'`. Pin.
14. **Missing required field** (e.g., no `confidence`): same. Pin.
15. **Markdown-wrapped JSON** (Gemini returns ` ```json\n{...}\n``` `): the validator strips common markdown fences before parsing; pin via fixture.
16. **Plain non-JSON text**: rejected. Pin via fixture (`aiqc-rejects-malformed`).

### Writeback semantics

17. **Text resolution**: `resolvedKind === 'text'` + `text === 'Quarterly Revenue'` → element's `text` field updated; if element type was `ShapeElement` (no `text` field) it converts to `TextElement` with the resolved text. Pin via fixture.
18. **Shape resolution**: `resolvedKind === 'shape'` + `shapeKind === 'rounded-rect'` → element's `shape` field becomes `'roundRect'` (canonical name). Pin via fixture (`aiqc-resolves-shape`).
19. **Fill color**: `fillColor: '#FF0000'` → element's fill updated. Pin.
20. **`inheritsFrom` preserved**: an element with `inheritsFrom = { templateId, placeholderIdx }` retains it after Gemini-resolution. Pin via fixture.
21. **Removed pendingResolution entry**: a resolved element's entry is dropped from `tree.pendingResolution`; the array shrinks accordingly. Pin.

### Cost cap

22. With `maxCallsPerDeck: 2` and 5 residuals, exactly 2 Gemini calls fire (the LLM mock records call count). The first 2 in `pendingResolution` order are attempted; the remaining 3 get `outcome: 'skipped-cap'` and emit `LF-GSLIDES-LOW-MATCH-CONFIDENCE` each. Pin via fixture (`aiqc-cap-hit`).
23. **Deck-level cap-hit flag**: `LF-GSLIDES-AI-QC-CAP-HIT` emits exactly once when any residual was skipped due to cap. Pin.
24. The cap doesn't apply per slide; a single slide with 100 residuals on a deck with a default cap of 100 would consume the entire budget. Pin via fixture (single slide × 5 residuals × cap 3 → 2 skipped on the same slide).
