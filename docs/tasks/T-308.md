## Architectural decisions

### D-T308-1. Seven invariants from ADR-004 Â§D6

Each invariant maps to a check function:

| # | Invariant | Function | Severity |
|---|---|---|---|
| 1 | Valid frontmatter | `checkFrontmatter(preset)` | error |
| 2 | `clipKind` in registry | `checkClipKindExists(preset)` | error |
| 3 | Bespoke-font preset has `fallbackFont` | `checkBespokeFontHasFallback(preset, fontRegistry)` | error |
| 4 | Interactive clip has non-empty `staticFallback` | `checkInteractiveStaticFallback(preset)` | error |
| 5 | Cluster A/B/D/F/G has `signOff.typeDesign` populated | `checkTypeDesignSignOff(preset)` | error |
| 6 | `signOff.parityFixture` populated | `checkParityFixtureSignOff(preset)` | error (warning until cluster merge) |
| 7 | `source` resolves in compass | `checkCompassAnchor(preset, compassDoc)` | error |

Each function returns `{ ok: true } | { ok: false, message: string }`. The script aggregates violations and reports them all (do not fail-on-first).

### D-T308-2. Cluster sign-off scoping (invariant 5 + 6)

Per ADR-004 Â§D4, only **clusters A (news), B (sports), D (titles), F (captions), G (CTAs)** require type-design-consultant batch sign-off. Clusters C (weather), E (data), H (AR) do not (no bespoke fonts at the cluster level).

| C | `weather` |
| D | `titles` |
| E | `data` |

Invariant 5 is conditional on cluster âˆˆ {A, B, D, F, G} (i.e., `news`, `sports`, `titles`, `captions`, `ctas`).

### D-T308-3. Bespoke-font detection (invariant 3)

Per ADR-004: a "bespoke font" is one whose preferred-font license is `proprietary-byo` OR `commercial-byo` (per the T-307 vocabulary).

```ts
const BESPOKE_LICENSE_ATOMS = new Set<FontLicenseAtom>(['proprietary-byo', 'commercial-byo']);

function isBespokeFont(preferredFont): boolean {
  const expr = parseFontLicenseExpression(preferredFont.license);
  return expr.atoms.some(atom => BESPOKE_LICENSE_ATOMS.has(atom));
}
```

Invariant 3 fails if `isBespokeFont(preferredFont) && !preset.fallbackFont`.

### D-T308-4. Compass-anchor resolution (invariant 7)

Compass file: `docs/compass.md` (per Phase 13 scaffold). Preset's `source` field is a relative URL with an anchor: `compass.md#cluster-news--cnn-classic`.

Anchor resolution rule:

1. Parse the URL into `{path, anchor}`.
2. Verify the file at the workspace-root-relative path exists.
3. Verify the anchor is a present heading slug in the file (case-insensitive, hyphen-spaced match).
4. If the source is an external URL (`https://...`), accept without verification (escalation: external sources need a separate validator).

If `docs/compass.md` doesn't exist yet (Phase 13 scaffold may not have shipped it), T-308 should NOT throw on missing-compass â€” instead, emit a single one-time warning and SKIP invariant 7 globally. Allows T-308 to ship before the compass.

### D-T308-5. `clipKind` registry (invariant 2)

The "clip registry" referenced in ADR-004 Â§D6 invariant 2 is the set of valid `clipKind` strings a preset may declare. Today this is enumerated at `packages/schema/src/elements/index.ts` via the `Element` discriminated union.

Build the valid set at script-init time:

```ts
const VALID_CLIP_KINDS = new Set([
  'text', 'image', 'shape', 'group', 'table',
  'blender-clip', 'interactive-clip',
  // Plus the family names from interactive: 'shader', 'three-scene', etc. â€” clipKind for interactive presets references the family.
]);
```

If a preset's `clipKind` doesn't match, invariant 2 fails.

**Spec call**: T-308 ships the set hardcoded in script header (mirroring `check-licenses.ts`'s style for SPDX atoms). Future tasks adding new clip kinds update this set.

### D-T308-6. Determinism + browser-bundle posture

`scripts/check-preset-integrity.ts` is a Node script. It uses `fs`, `path`, `tsx` runtime. Determinism scan does not apply. Browser-bundle hazard does not apply (it's a script, not a package).

The script imports `@stageflip/schema/presets/node` (Node-only subpath; T-304 + T-307 contributions).

## Acceptance criteria

### Per-invariant checks

1. **Invariant 1 (frontmatter)**: synthetic preset with malformed frontmatter (e.g., missing required field) fails. Pin via fixture.
2. **Invariant 2 (clipKind)**: synthetic preset with `clipKind: 'mystery'` fails. Pin.
3. **Invariant 3 (bespoke fallback)**: synthetic preset with `preferredFont.license: 'proprietary-byo'` and no `fallbackFont` fails. Pin.
4. **Invariant 3 negative**: same preset WITH `fallbackFont` passes.
5. **Invariant 4 (interactive staticFallback)**: synthetic interactive-family preset with empty `staticFallback` fails. (Note: this overlaps T-305's schema-level check â€” T-308 catches it at preset-gate level.)
6. **Invariant 5 (typeDesign sign-off, A/B/D/F/G)**: synthetic preset in `news` cluster with `signOff.typeDesign: 'pending-cluster-batch'` PASSES. With `signOff.typeDesign` field MISSING (would fail T-304 schema, but pin defense-in-depth).
7. **Invariant 5 negative**: synthetic preset in `weather` cluster (cluster C) without `signOff.typeDesign` PASSES â€” invariant 5 doesn't apply to weather.
8. **Invariant 6 (parityFixture)**: emits warning (not error) when `signOff.parityFixture: 'pending-user-review'`. Pin via captured stdout.
9. **Invariant 7 (compass anchor)**: synthetic preset with `source: 'docs/compass.md#cnn-classic'` resolves IF the compass file + anchor exists. Synthetic with non-existent anchor fails.
10. **Invariant 7 missing-compass**: when `docs/compass.md` is absent (script-time check), invariant 7 is SKIPPED with a warning, not failed.

### Script integration

11. `pnpm check-preset-integrity`: runs the script via tsx. Exits 0 on PASS, 1 on FAIL.
12. **Output format**: matches `check-licenses.ts` style (per-check progress lines + final PASS/FAIL).
13. **Aggregates violations**: 3 errors across 3 invariants surface in a single output, not first-error-only.
14. **PASS at HEAD**: against the actual on-disk presets, the script PASSES (or PASSES-WITH-WARNINGS).

### Performance

15. Total runtime < 10s for the current 50-preset corpus. Pin via timing.

### Backward compatibility

16. The full workspace test suite passes unchanged.
17. `check-licenses` + `check-determinism` + `check-skill-drift` not modified.
