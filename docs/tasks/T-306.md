# T-306 — `packages/runtimes/interactive/` runtime tier

**Branch**: `task/T-306-runtime-tier-skeleton`

## Goal

Stand up the **interactive runtime tier package** per ADR-003 §D1 + §D4. T-306 ships:

1. **`packages/runtimes/interactive/`** package — the runtime that mounts `liveMount` for HTML / live-presentation / display-interactive / on-device-player exports.
2. **`PermissionShim`** — mount-time gate that requests browser permissions per the clip's declared envelope; falls back to `staticFallback` on denial.
3. **`InteractiveMountHarness`** — programmatic API for editors / renderers to mount + unmount + dispose live clips.
4. **Contract tests** — generic test suite any frontier clip family (Phase γ) must pass.
5. **Determinism-scope exemption** — `scripts/check-determinism.ts` excludes `packages/runtimes/interactive/**` per ADR-003 §D5. (T-309 ships the shader sub-rule that DOES apply inside the tier; T-306 ships the basic exemption only.)
6. **`InteractiveClipRegistry`** — frontier-family discriminator → component-loader map. Phase γ clips register here.

T-306 does NOT ship any specific frontier clip implementation (those are Phase γ tasks T-340+). T-306 ships the **infrastructure** the clips plug into.

## Dependencies

- **Required reading**: ADR-003 (interactive runtime tier) — D1/D4/D5 lock this scope.

### D-T306-2. `MountContext` shape

```ts
export interface MountContext {
  /** The InteractiveClip schema instance being mounted. */
  clip: InteractiveClip;
  /** DOM node to mount into. */
  root: HTMLElement;
  /** Permission shim already vetted — guarantees these are GRANTED. */
  permissions: ReadonlyArray<Permission>;
  /** Tenant policy for feature-flag checks (ADR-005 §D3). Permissive default in T-306. */
  tenantPolicy: TenantPolicy;
  /** Telemetry hook (T-264 OpenTelemetry / observability). */
  emitTelemetry: (event: string, attributes: Record<string, unknown>) => void;
  /** AbortSignal for unmount cleanup. */
  signal: AbortSignal;
}

export interface MountHandle {
  /** Update props on a live mount (e.g., user edits the clip's params). */
  updateProps(props: Record<string, unknown>): void;
  /** Imperative dispose (separate from AbortSignal — useful for editor unload). */
  dispose(): void;
}

export type ClipFactory = (ctx: MountContext) => Promise<MountHandle>;
```

The `signal: AbortSignal` is canonical across StageFlip async APIs (T-261 presence subscribe, T-270 storage subscribeUpdates). Reuse the pattern.

### D-T306-3. Permission shim flow

Per ADR-003 §D4:

```
PermissionShim.mount(clip):
  1. tenantPolicy.canMount(clip.family)
       → false: short-circuit to staticFallback rendering
  2. for each permission in clip.liveMount.permissions:
       browser.permissions.query/request via the appropriate API
         (mic: getUserMedia({audio:true}); camera: getUserMedia({video:true}); network: noop — assumed always granted)
       → denied: short-circuit to staticFallback + emitTelemetry('permission-denied', {permission, family})
  3. all granted: resolve to MountContext with permissions array
```

The shim is a stateful object (one per editor session) that caches granted permissions for the session. Avoids prompting the user multiple times for the same clip family.

### D-T306-4. Registry shape

```ts
export class InteractiveClipRegistry {
  private factories = new Map<InteractiveFamily, ClipFactory>();
  register(family: InteractiveFamily, factory: ClipFactory): void;
  resolve(family: InteractiveFamily): ClipFactory | undefined;
  list(): InteractiveFamily[];
}

export const interactiveClipRegistry = new InteractiveClipRegistry();
```

Module-level singleton. Phase γ tasks call `interactiveClipRegistry.register('shader', shaderFactory)` at clip-package import time.

T-306 ships an empty registry; the shipped families list is `[]`. Tests use a stub factory.

### D-T306-5. Determinism-scope exemption

`scripts/check-determinism.ts` is amended to exclude `packages/runtimes/interactive/**` from its source walk. The exemption is documented inline in the script with a comment citing ADR-003 §D5.

T-306 ships ONLY this package-level exemption. T-309 lands the shader sub-rule that re-applies determinism inside the tier (`uFrame`-only uniform updaters). Per the spec note: don't pre-implement T-309's logic in T-306.

### D-T306-6. Contract test suite

The runtime tier defines a `contractTestSuite(factory: ClipFactory)` that any frontier clip family must pass:

- mount produces a `MountHandle`.
- `updateProps` triggers re-render (or whatever the clip's contract says — verifiable via the family's own assertions).
- `dispose()` cleans up DOM + event listeners.
- `signal.abort()` triggers same cleanup as `dispose()`.
- Permission denial during mount surfaces the `staticFallback` path.

Each Phase γ clip task imports `contractTestSuite` and runs it against its factory. T-306 ships the suite + a stub factory that exercises every assertion.

### D-T306-7. Browser-bundle posture

`packages/runtimes/interactive/**` is consumed by the browser apps (it's literally the browser runtime tier). It must NOT import Node-only modules (`fs`, `path`, `child_process`).

The package depends on `react` (mount harness uses React 18+ root API) and `@stageflip/schema` (the browser-safe surface, not `@stageflip/schema/presets/node`).

### D-T306-8. Determinism posture

By ADR-003 §D5, this tier is OUT of scope for `check-determinism`. The package can use:

- `Date.now()`, `performance.now()`, `Math.random()`
- `setTimeout`, `setInterval`, `requestAnimationFrame`
- `fetch`, `XMLHttpRequest`, `MediaRecorder`, `getUserMedia`
- `Worker`, `SharedWorker`

But: T-309's shader sub-rule will re-impose `uFrame`-only on `ShaderClip` and `ThreeSceneClip` factories. T-306 does NOT ship those clips, so the sub-rule isn't yet enforceable. Just be aware.

## Acceptance criteria

Each AC has a Vitest test, written first and failing.

### Contract types (`contract.ts`)

1. `MountContext`, `MountHandle`, `ClipFactory`, `TenantPolicy` types exported per D-T306-2.
2. **Type-level test**: a `ClipFactory` instance can be defined and called with a valid `MountContext`. Pin via TS exhaustive type-check.

### Permission shim (`permission-shim.ts`)

3. `PermissionShim.mount(clip, root)`: when `clip.liveMount.permissions: []`, succeeds without prompting.
4. With `permissions: ['network']`: succeeds without prompting (network is a no-op permission per D-T306-3).
5. With `permissions: ['mic']`: calls `getUserMedia({ audio: true })`; on grant returns `{ granted: true, ...ctx }`; on denial returns `{ granted: false, fallbackTo: 'static' }`.
6. With `permissions: ['camera']`: calls `getUserMedia({ video: true })`; same grant/deny semantics.
7. **Tenant-policy gate**: `tenantPolicy.canMount(family)` returning `false` short-circuits BEFORE any browser prompt; falls back to static.
8. **Permission caching**: a second mount for the same family within the session does NOT re-prompt.
9. `emitTelemetry('permission-denied', ...)` called on every denial. Pin via spy.

### Registry (`registry.ts`)

10. `interactiveClipRegistry.register('shader', factory)`: registers; subsequent `resolve('shader')` returns the same factory.
11. **Re-registration**: registering a second factory for the same family throws (avoid silent shadowing).
12. `resolve('unknown-family')`: returns `undefined`.
13. `list()`: returns sorted family names.
14. **Module-level singleton**: importing `interactiveClipRegistry` from two different modules returns the SAME instance (verify via reference equality).

### Mount harness (`mount-harness.ts`)

15. `InteractiveMountHarness.mount(clip, root, signal)`:
   - calls `permissionShim.mount(clip, root)` first.
   - on grant: resolves the registered factory for `clip.liveMount.family`; calls factory with constructed `MountContext`; returns the resulting `MountHandle`.
   - on denial: renders `staticFallback` via `fallbackRendering.render(...)` instead.
16. **Factory not registered**: `family` not in registry → throws `InteractiveClipNotRegisteredError`.
17. **AbortSignal cleanup**: aborting the signal triggers `MountHandle.dispose()`.
18. **Idempotent dispose**: calling `dispose()` twice is a no-op.

### Fallback rendering (`fallback-rendering.ts`)

19. `renderStaticFallback(elements, root)`: renders the canonical-element array into the root via React render. Each element type maps to a DOM representation (text → `<span>`, image → `<img>`, etc.) — match the existing renderer-core conventions OR document the divergence.
20. Handles empty arrays gracefully (`elements: []` is rejected by T-305 schema, but the function should be defensive — no throw, just empty render).

### Contract test suite (`contract-tests/index.ts`)

21. `contractTestSuite(factory: ClipFactory)`: a function that returns a Vitest `describe` block. Each Phase γ family imports + invokes this suite against its own factory.
22. The suite asserts: mount returns a `MountHandle`; `updateProps` callable; `dispose()` cleans up; `signal.abort()` cleans up; permission denial returns the static path.
23. T-306 includes a stub factory + tests that pass against the stub. This validates the suite itself.

### Determinism exemption

24. `scripts/check-determinism.ts` excludes `packages/runtimes/interactive/**`. Verify by reading the script + running `pnpm check-determinism` (which should now PASS even with `Date.now()` calls in `permission-shim.ts`, etc.).

### Backward compatibility

25. The full workspace test suite passes unchanged. Adding the determinism exemption does NOT shrink the rule's scope on other paths.
26. `packages/frame-runtime/**`, `packages/runtimes/blender/**` — neither modified; both still pass `check-determinism`.

## Escalation triggers (CLAUDE.md §6)

- The fallback-rendering implementation needs to call into `@stageflip/renderer-core` to share render code with frame-runtime, but importing the renderer-core surface introduces circular deps or browser-bundle hazards. Escalate; we may need a small shared "static-element-to-DOM" helper extracted to a third package.
- The React 18+ root API conflicts with how editor apps already mount. Escalate; the harness might need to accept an existing root rather than create one.
- Permission API differences between Chromium and Safari surface (e.g., Safari refuses `permissions.query({name: 'microphone'})`). Escalate; we may need a feature-detection wrapper.
- `getUserMedia` requires a user gesture to prompt; tests may not be able to simulate this. Escalate; mocking the permission flow is acceptable but the production code path needs a real-browser smoke test that lands separately.
