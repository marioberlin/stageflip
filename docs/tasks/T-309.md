# T-309 — Extended `check-determinism` — shader sub-rule

**Branch**: `task/T-309-shader-sub-rule`

## Goal

Extend `scripts/check-determinism.ts` with a **shader sub-rule** that re-applies determinism inside the interactive tier (which T-306 exempted at the package level), per ADR-003 §D5 + ADR-005 §D2.

The interactive tier (`packages/runtimes/interactive/**`) is broadly exempt from the §3 determinism rules. BUT: shaders and Three.js scenes are **frame-deterministic by construction** within that tier. T-309 enforces this:

- Uniform-updater functions for `ShaderClip` and `ThreeSceneClip` must accept `frame` as their sole time-like input.
- Reading `performance.now()` or `Date.now()` inside a uniform updater is a lint error.

T-309 lands BEFORE any `ShaderClip` implementation (those are Phase γ tasks T-340+) so the rule is enforced from day one.

## Architectural decisions

### D-T309-1. Sub-rule scope: matched paths

The sub-rule applies to files matching:

```
packages/runtimes/interactive/src/clips/shader/**/*.ts
packages/runtimes/interactive/src/clips/three-scene/**/*.ts
```

Plus any file (across the repo) that **exports** a function decorated with `@uniformUpdater`. The decorator approach is forward-compat — Phase γ implementers tag their uniform updaters, and T-309's lint follows the tags wherever they live.

### D-T309-2. The forbidden-API set

Inside a uniform-updater function body (statically-detected via decorator OR matched path), these APIs are forbidden:

- `performance.now()`
- `Date.now()`
- `new Date()` (without arg)
- `Math.random()`
- `setTimeout` / `setInterval`
- `requestAnimationFrame` / `cancelAnimationFrame`

The function MUST accept `frame: number` (or any positional `frame` named parameter) as a parameter. Pin via AST inspection.

### D-T309-3. Detection mechanism

Two strategies, T-309 ships both:

1. **Path-based**: `packages/runtimes/interactive/src/clips/{shader,three-scene}/**/*.ts` — every file in these directories has its top-level exported functions checked against the forbidden-API set.
2. **Decorator-based**: any function annotated with a `@uniformUpdater` JSDoc tag (e.g., `/** @uniformUpdater */ function uFrame(frame) { ... }`) is checked, regardless of path.

This is the "dual gate" — path catches families T-309 names; decorator catches future families that may not be in those paths yet.

### D-T309-4. Implementation: AST-based via TypeScript compiler API

The current `check-determinism.ts` is regex-based. T-309 keeps the regex path for the broad exemption logic (still zero-cost) but introduces an **AST-pass** for the shader sub-rule:

```ts
import * as ts from 'typescript';

function checkUniformUpdaterFunction(node: ts.FunctionDeclaration | ts.ArrowFunction, sourceFile: ts.SourceFile): Violation[] {
  // Walk the function body. For each Identifier or PropertyAccess matching the forbidden-API set, emit a violation.
}
```

Why TypeScript's compiler API: (a) it's already in workspace; (b) a regex on `Date.now` would over-match (e.g., a string literal `"Date.now() returns ms"` in a comment), causing false positives; (c) AST-based gives precise file:line:column for violations.

### D-T309-5. Performance budget

The existing `check-determinism` runs in <1s on the current source set. T-309's AST pass adds work ONLY for files matching the sub-rule paths (~few files when Phase γ ships, none today). Total runtime budget: <2s incremental for the sub-rule. If exceeded, optimize.

### D-T309-6. Output format

Match existing `check-determinism.ts` output. New section appended:

```
check-determinism: scanned 68 source files
check-determinism [§3 broad rule]: PASS
check-determinism [shader sub-rule]: PASS (0 uniform-updaters detected)

check-determinism: PASS
```

When a violation surfaces:

```
check-determinism [shader sub-rule]: FAIL
  - packages/runtimes/interactive/src/clips/shader/foo.ts:42:13
    forbidden API `performance.now()` in uniform-updater `uFrame`
    (use `frame` parameter instead — ADR-003 §D5)

check-determinism: FAIL (1 violation)
```

### D-T309-7. Determinism + browser-bundle posture

Script is Node-only; `fs`/`path`/`typescript` allowed. No browser-bundle implications.

The `typescript` package is already a workspace dep (used by other build steps). Verify on whitelist (Apache-2.0).

## Acceptance criteria

### Sub-rule detection

1. **Path-based, positive case**: a file at `packages/runtimes/interactive/src/clips/shader/foo.ts` with `function uFrame(frame: number) { return Date.now(); }` causes the sub-rule to FAIL with line:column.
2. **Path-based, negative case**: same path with `function uFrame(frame: number) { return frame * 0.001; }` PASSES.
3. **Decorator-based, positive case**: any file with `/** @uniformUpdater */ function foo(frame) { performance.now(); }` FAILS.
4. **Decorator-based, negative case**: same shape with no decorator PASSES the sub-rule (broad rule may still flag; that's separate).
5. ~~**Frame-only signature**: `function uFrame(frame: number) { return frame; }` PASSES.~~ **Superseded by T-309a** — the missing-frame check is dropped entirely; a frame-only signature still PASSES (no forbidden API), but the rule no longer cares whether `frame` is declared. See `docs/tasks/T-309a.md`.
6. ~~**Missing frame parameter**: `function uFrame() { return Date.now(); }` FAILS twice (no frame param + Date.now usage). Both surface.~~ **Superseded by T-309a** — `function uFrame() { return Date.now(); }` now FAILS with ONE violation (the API call). The "missing-frame-parameter" violation kind is removed. See `docs/tasks/T-309a.md` AC #6 → #8.

### Forbidden-API coverage

7. Each of the 6 forbidden APIs (per D-T309-2) triggers a sub-rule violation in a positive-case fixture. Pin all 6.

### Output format

8. Output line `check-determinism [shader sub-rule]: PASS (N uniform-updaters detected)` when no violations; the count is the number of inspected functions.
9. Violation lines show file:line:column + the offending API + a hint string.
10. Exit 0 on PASS; 1 on FAIL.

### Performance

11. **Sub-rule runtime <2s**: with N=10 fixture files containing uniform-updaters, total sub-rule runtime <2s. Pin via timing.
12. **Existing rule unchanged**: regex-based broad rule's runtime not regressed (compare before/after).

### Backward compatibility

13. Existing `check-determinism` behavior on the broader source set unchanged — every file flagged before is still flagged; no false positives introduced.
14. The full workspace test suite passes unchanged.

### Coverage + gates

15. Coverage on `scripts/check-determinism.ts` ≥85% (combined regex + AST paths).
16. `pnpm typecheck`, `pnpm lint`, `pnpm test`, `pnpm check-licenses`, `pnpm check-remotion-imports`, `pnpm check-determinism`, `pnpm check-skill-drift`, `pnpm size-limit`, `pnpm check-preset-integrity` all green.
