# T-250 — workflow-skills sweep (import-*, export-*, references/ tier wiring)

**Branch**: `task/T-250-workflow-skills-sweep`

## Goal

Sweep across the workflow-tier skill files in `skills/stageflip/workflows/**/SKILL.md` to document patterns that have crystallized during Phase 11 implementation. Plan-row at `docs/implementation-plan.md:540` extends the original scope to cover:

1. **Document the API-geometry → render-pixel conversion math** ("rendered PNG is the source of truth" principle from T-244 §3).

3. **Document native grouping handling** (groups, tables, placeholder inheritance per T-251).
4. **Document the loss-flag taxonomy across importers** (now ~30+ codes across pptx / gslides / hyperframes-html / pptx-export / gslides-export / design-system sources).

T-250 is a **docs-only sweep** — no source code changes. Workflow skills are agent-readable how-to docs that capture cross-cutting patterns importers + exporters share, NOT per-package reference material (which lives in `skills/stageflip/reference/<package>/` per the existing convention).

### 2. Cross-cutting "API-geometry → render-pixel" doc

Lives at `skills/stageflip/workflows/import-google-slides/SKILL.md` (already exists; updated by T-244 implementer to `status: substantive`).

**Append/extend, NOT rewrite** (per PR #215 Reviewer): T-244 implementer already added "rendered PNG is source of truth" framing, pixel-candidate pipeline overview, and `CvCandidateProvider` split. T-250's role is incremental gap-fills — verify what's already there, then add the explicit `emu_to_px` math + the `runAiQcConvergence` cite. Do NOT rewrite existing prose.

T-250 augments the existing skill with explicit math + diagram:

```
emu_to_px = renderSize / pageSizeEmu      // per-axis
elementBboxPx = applyAffineToUnitSquare(transform * emu_to_px) ∩ slideBoundsPx
```

Plus the "rendered PNG is the source of truth" principle: API geometry alone isn't pixel-accurate (server-side autofit, padding inferred differently, theme tokens resolve to different actual colors); pairing with the rendered thumbnail recovers ground truth.

### 3. Cross-cutting "pixel-candidate pipeline" doc

Same skill file. Documents the deterministic-CV-first / Gemini-second principle:

- ~90% of objects resolve in the cheap deterministic pass (PaddleOCR text-content match, OpenCV findContours / connectedComponentsWithStats, optional SAM 2 mask containment, z-order plausibility).
- ~10% residual reach `runAiQcConvergence` (T-246) for a single Gemini multimodal call per residual.
- The CV provider behind `CvCandidateProvider` is pluggable (Stub for tests, HTTP for production via T-244-cv-worker's Cloud Run service).

### 4. Cross-cutting "native grouping" doc

Lives at `skills/stageflip/concepts/schema/SKILL.md` (existing) — referenced from `workflows/import-pptx/SKILL.md` and `workflows/import-google-slides/SKILL.md`. **`inheritsFrom` + `applyInheritance` are already documented** in this skill (line 34 per PR #215 Reviewer).
