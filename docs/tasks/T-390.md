# T-390 — `AiChatClip` `staticFallback` (captured transcript)

**Branch**: `task/T-390-ai-chat-clip-static-fallback`

## Goal

Ship the `staticFallback` for `family: 'ai-chat'`. When the network permission gate denies (rare in v1's no-op grant world but possible after T-403's tenant allowlists land), or the tenant policy refuses, or an export pipeline targets a static medium (MP4/PPTX/PDF), the mount-harness routes to a deterministic transcript: the `systemPrompt` summary + a captured turn sequence supplied at authoring time.

T-390 is the `staticFallback` half of ADR-005 §D1's `AiChatClip` row. After T-390 merges, `family: 'ai-chat'` is structurally complete.

T-390 registers `aiChatStaticFallbackGenerator` against the **T-388a static-fallback generator registry** (already on `main`), not via a family-hardcoded harness branch.

## Architectural decisions

### D-T390-1. Schema addition — `capturedTranscript?` on `aiChatClipPropsSchema`

T-389 shipped `aiChatClipPropsSchema` with `posterFrame` but no transcript field. T-390 adds:

```ts
// packages/schema/src/clips/interactive/ai-chat-props.ts (MODIFIED)
export const aiChatClipPropsSchema = z.object({
  // ... existing T-389 fields ...
  /** Pre-captured turn sequence for the static fallback. Optional; absent → systemPrompt-only summary. */
  capturedTranscript: z.array(z.object({
    role: z.enum(['user', 'assistant']),
    text: z.string().min(1),
  })).optional(),
}).strict();
```

Browser-safe. Existing T-389 fixtures continue to validate (the field is optional).

### D-T390-2. Generator output — `TextElement` per turn + optional system-prompt prefix

`defaultAiChatStaticFallback` returns:

1. A `TextElement` rendering the `systemPrompt` (truncated to N chars — say 200 — if longer; documented).
2. One `TextElement` per turn in `capturedTranscript`, alternating role labels via data attributes (`data-role="user" | "assistant"`).
3. Each TextElement's transform is laid out vertically within the clip's bounds (line-spacing logic; deterministic).

When `capturedTranscript` is absent, returns just the `systemPrompt` element + a single placeholder TextElement marked `data-role="placeholder"` whose text is supplied via the host through `posterText`-style props (NOT in this spec — host controls via app-level i18n; T-390 ships a single `data-role="placeholder"` element with empty text by default).

### D-T390-3. Determinism

`defaultAiChatStaticFallback` is subject to the broad §3 rule (no rAF, no random, no time). Pure transformation over inputs. Same posture as `defaultVoiceStaticFallback` (T-388).

### D-T390-4. Telemetry

`ai-chat-clip.static-fallback.rendered` (attributes: `reason`, `width`, `height`, `transcriptTurnCount`, `systemPromptLength` — integer lengths only, NOT bodies; same privacy posture as T-389 D-T389-8).

### D-T390-5. Registration via T-388a registry

`packages/runtimes/interactive/src/clips/ai-chat/index.ts` gains a side-effect call:

```ts
import { staticFallbackGeneratorRegistry } from '../../static-fallback-registry.js';
import { aiChatStaticFallbackGenerator } from './static-fallback.js';

interactiveClipRegistry.register('ai-chat', aiChatClipFactory);  // existing T-389 line
staticFallbackGeneratorRegistry.register('ai-chat', aiChatStaticFallbackGenerator);  // NEW
```

The harness's family-agnostic dispatch (T-388a) picks up the registration without further changes.

## Acceptance criteria

### Schema

1. `aiChatClipPropsSchema` accepts `capturedTranscript` of any non-empty array of `{role, text}`.
2. `capturedTranscript: [{ role: 'user', text: '' }]` → schema error (text min 1).
3. `capturedTranscript: [{ role: 'invalid', text: 'x' }]` → schema error (role enum).
4. Existing T-389 fixtures (without `capturedTranscript`) continue to validate.

### Generator determinism

5. `defaultAiChatStaticFallback({ width: 640, height: 360, systemPrompt: 'p', capturedTranscript: [{role:'user',text:'hi'}] })` returns the same Element[] byte-for-byte across two calls.
6. Different `systemPrompt` → different markup (text differs).
7. Empty `capturedTranscript` → systemPrompt + single placeholder element only.

### Registry integration

8. After importing `clips/ai-chat`, `staticFallbackGeneratorRegistry.resolve('ai-chat')` returns the generator.
9. The harness routes empty `staticFallback` for `family: 'ai-chat'` clips to the generator (verified end-to-end via mount-harness test mirroring T-388 AC #12).
10. Authored `staticFallback` (non-empty) wins; the generator is still called with `reason: 'authored'` for telemetry (T-388a D-T388a-3 contract).

### Bounds

11. Each TextElement's transform fits within the clip's bounds (no element extends past `width × height`).
12. Vertical layout: turn N's transform.y is greater than turn N-1's (deterministic stacking).

### Telemetry

13. `ai-chat-clip.static-fallback.rendered` emits with documented attribute shape; `transcriptTurnCount` and `systemPromptLength` are integers, NOT the bodies.

### Coverage + gates

14. Coverage on every new file ≥ 85 %.
15. `pnpm typecheck`, `pnpm lint`, `pnpm test`, `pnpm check-licenses`, `pnpm check-remotion-imports`, `pnpm check-determinism`, `pnpm check-skill-drift`, `pnpm gen:tool-skills:check`, `pnpm check-preset-integrity`, `pnpm size-limit` all green.
