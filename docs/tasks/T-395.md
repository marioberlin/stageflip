# T-395 — `AiGenerativeClip` `liveMount` (playback-time generation)

**Branch**: `task/T-395-ai-generative-clip-live-mount`

## Goal

Ship the fifth γ-live family: `AiGenerativeClip` `liveMount`. A schema-validated `InteractiveClip` with `family: 'ai-generative'`, registered in `interactiveClipRegistry`, mountable via the T-306 `ClipFactory` contract. At mount time the factory feeds `liveMount.props.prompt` to a host-injected `AiGenerativeProvider`, awaits the generated artifact (a `Blob` + `contentType`), and renders it into an `<img>` element under the supplied root. The clip exposes a typed `MountHandle` with `regenerate()`, `getResult()`, `onResult()`, and `onError()`; emits typed lifecycle telemetry whose attributes carry **integer lengths only** (the prompt body never appears).

T-395 is the **fifth application of the second γ pattern** (event-driven, no §3 reuse). After T-395 + T-396 land, all five γ-live family pairs ship and the family catalogue ADR-005 §D1 is fully covered (modulo modality follow-ups documented below).

T-395 ships **`liveMount` only**, restricted to **image-modality** generation in v1. **T-396 ships `staticFallback`** (curated example output rendered as `ImageElement`) as a separate S-sized task. Audio / video / 3D modalities and authoring-time pre-bakes are deferred to Phase 14 (ADR-006 asset generation; the implementation-plan note at line 873 documents the boundary: "Distinct from the frontier `AiGenerativeClip` (ADR-005 §D1) which generates at playback time; this phase produces frozen files consumed by existing `MediaElement` schema slots").

## Architectural decisions

### D-T395-1. `AiGenerativeClip` lives entirely in `runtimes-interactive/src/clips/ai-generative/`

No new package; no §3 runtime to reuse. Same posture as T-387 / T-389 / T-391 / T-393. The clip is event-driven (one-shot generation + manual `regenerate()`), not frame-driven. Confirms the second γ pattern across five families.

### D-T395-2. Schema narrowing — `aiGenerativeClipPropsSchema`

```ts
// packages/schema/src/clips/interactive/ai-generative-props.ts (NEW)
export const aiGenerativeClipPropsSchema = z.object({
  /**
   * The generation prompt. Per-clip identity — baked into the schema.
   * Non-empty; the clip's identity is "what to generate."
   */
  prompt: z.string().min(1),
  /**
   * Provider name. Free-form (`'openai' | 'stability' | 'replicate' |
   * 'tenant-supplied' | ...`); the schema accepts any non-empty string
   * so tenant adapters extend without a schema bump. The host's
   * injected `AiGenerativeProvider` validates / dispatches on this.
   */
  provider: z.string().min(1),
  /**
   * Model identifier within the provider (`'dall-e-3'` /
   * `'stable-diffusion-xl'` / etc.). Non-empty.
   */
  model: z.string().min(1),
  /**
   * Optional negative prompt — concepts to avoid. Forwarded verbatim
   * to providers that support it (Stable Diffusion); ignored by
   * providers that don't (DALL-E). The clip does not normalize.
   */
  negativePrompt: z.string().optional(),
  /**
   * Optional deterministic seed. Forwarded verbatim; providers that
   * don't honour seeds (DALL-E) ignore it. Same-seed-same-output is
   * the provider's responsibility, not the clip's.
   */
  seed: z.number().int().optional(),
  /**
   * Optional output dimensions. Default to the clip transform's
   * dimensions when omitted. Some providers constrain to specific
   * values (DALL-E requires 256 / 512 / 1024 squares); the host's
   * provider adapter is responsible for validation / quantisation.
   */
  width: z.number().int().positive().optional(),
  height: z.number().int().positive().optional(),
  /**
   * Frame at which `staticFallback` (T-396 curated example) is
   * sampled. Convention reused from shader / three-scene / voice /
   * ai-chat / live-data / web-embed.
   */
  posterFrame: z.number().int().nonnegative().default(0),
}).strict();
```

Browser-safe (pure Zod). Mirrors the discriminator pattern from T-383/T-384/T-387/T-389/T-391/T-393.

**No `modality` field in v1** — see "Out of scope." When a second modality lands, the field is added with `z.enum(['image', 'audio', ...]).default('image')` — a non-breaking change for existing fixtures.

### D-T395-3. Permissions — `['network']`; pre-prompt off

The clip declares `permissions: ['network']`. Per the existing shim (T-385), `network` is a no-op grant in v1; tenant allowlists land later (T-403). Pre-prompt is OFF by default for `network`-only clips per the precedent established by T-389 / T-391 / T-393. Documented in the skill.

### D-T395-4. `liveMount` API surface

The factory mounts a single `<img>` element under `ctx.root`. No React tree (same posture as T-393 WebEmbed). When generation resolves, the factory sets `img.src` to a blob URL (`URL.createObjectURL`) so the renderer paints the artifact natively.

```ts
interface AiGenerativeClipMountHandle extends MountHandle {
  /**
   * Re-run generation with the current props. Aborts any in-flight
   * generation first. Resolves when the new generation settles.
   */
  regenerate(): Promise<void>;
  /**
   * Latest resolved result. `undefined` until the first generation
   * resolves successfully. The `blob` is owned by the clip — hosts
   * that want to outlive the clip MUST clone the Blob.
   */
  getResult(): { blob: Blob; contentType: string } | undefined;
  /** Subscribe to result events. Returns unsubscribe. */
  onResult(handler: (event: ResultEvent) => void): () => void;
  /** Subscribe to error events. Returns unsubscribe. */
  onError(handler: (event: ErrorEvent) => void): () => void;
}

type ResultEvent = {
  kind: 'resolved';
  blob: Blob;
  contentType: string;
  durationMs: number;
};

type ErrorEvent = {
  kind: 'generate-error' | 'aborted';
  message: string;
};
```

### D-T395-5. `AiGenerativeProvider` interface — host-injected client

```ts
// packages/runtimes/interactive/src/clips/ai-generative/ai-generative-provider.ts (NEW)
export interface AiGenerativeProvider {
  /** Generate one artifact. Resolves on success; rejects on transport / quota / safety failure. */
  generateOnce(args: {
    prompt: string;
    negativePrompt?: string;
    model: string;
    width?: number;
    height?: number;
    seed?: number;
    signal: AbortSignal;
  }): Promise<{ blob: Blob; contentType: string }>;
}
```

Two implementations ship with T-395:

1. `HostInjectedAiGenerativeProvider({ generator })` — wraps a host-supplied `Generator` callable that returns a `Blob`. The host's adapter is the seam where provider SDKs / API tokens / tenant moderation live. The clip itself never touches `globalThis.fetch`.
2. `InMemoryAiGenerativeProvider({ scripted })` — for tests; resolves a scripted `Record<promptHash, ScriptedResult>`. Used by the factory tests; production code never instantiates.

A `@stageflip/ai-generative-abstraction` package mirroring `@stageflip/llm-abstraction` is a future asset-gen task — Phase 14 ADR-006 covers the pattern. T-395 ships only the seam.

### D-T395-6. No convergence test, no `frameSource` dependency

Confirms second γ pattern. The factory does NOT throw `MissingFrameSourceError`; it ignores `MountContext.frameSource`. Pin via test (mounting with `frameSource: undefined` succeeds).

### D-T395-7. Resource cleanup on dispose

`MountHandle.dispose()` MUST:

1. `signal.abort()` on any in-flight `generateOnce` call.
2. **Revoke the blob URL** (`URL.revokeObjectURL`) — without this the browser holds the blob indefinitely (memory leak; one slide × one clip × one regenerate = ~200KB held forever).
3. Drop the resolved-result reference (no leaked references for GC).
4. Unsubscribe all `onResult` / `onError` handlers.
5. **Detach the `<img>` element** from the DOM.
6. Be idempotent.

The blob-URL revocation step is load-bearing — `URL.createObjectURL` is the only browser API that creates a hidden GC root not handled by ordinary DOM teardown.

### D-T395-8. Telemetry — lifecycle + generation events

- `ai-generative-clip.mount.start` (attributes: `family: 'ai-generative'`, `provider`, `model`, `promptLength` integer — never the prompt body)
- `ai-generative-clip.mount.success`
- `ai-generative-clip.mount.failure` (reasons: `'invalid-props'`, `'generator-unavailable'`, `'permission-denied'`)
- `ai-generative-clip.generate.started` (attributes: `requestId`, `promptLength` integer)
- `ai-generative-clip.generate.resolved` (attributes: `requestId`, `durationMs`, `blobByteLength` integer, `contentType`)
- `ai-generative-clip.generate.error` (attributes: `requestId`, `errorKind`)
- `ai-generative-clip.dispose`

**Privacy posture**: prompt body, negativePrompt body, and generated blob bytes NEVER appear in telemetry attributes. `promptLength` and `blobByteLength` are integers only. Same posture as T-389 D-T389-8 / T-391 D-T391-8 / T-393 D-T393-8. Pin via test (AC #18).

### D-T395-10. Pattern-evaluation — fifth application reaffirms the T-393 ruling

T-393 D-T393-10 reaffirmed the T-391 D-T391-10 ruling at four families. T-395 is the **fifth** standalone-pattern application. The five γ-live shapes now span:

- `TranscriptionProvider.start({ stream, language, partial, onTranscript })` — streaming + discriminated-union callback.
- `LLMChatProvider.streamTurn({ ..., onToken, signal })` — streaming + per-token callback.
- `LiveDataProvider.fetchOnce({ ..., signal })` — request/response, text-shaped output.
- **No seam** for `WebEmbedClip` — the iframe is the transport.
- `AiGenerativeProvider.generateOnce({ ..., signal })` — request/response, **binary-Blob-shaped output**.

The fifth shape is structurally similar to LiveData (request/response + signal) but with a different output type (`Blob` vs `string`). A generic `ProviderSeam<T>` over five shapes (including "no seam") remains incoherent. Skill update reaffirms — "Five frontier-clip implementations now span four streaming/request shapes plus one no-seam case. The convention 'host supplies the client' is honoured at four of five; the fifth (web-embed) has no client to inject. No abstraction is extracted."

## Acceptance criteria

1. `aiGenerativeClipPropsSchema` parses a valid payload (prompt, provider, model, negativePrompt, seed, width, height, posterFrame).

3. Empty `provider` / empty `model` → schema error.

### Factory + registry

8. Re-importing throws `InteractiveClipFamilyAlreadyRegisteredError`.
9. Mount with `MountContext.frameSource: undefined` succeeds — no `MissingFrameSourceError`.

### Generation lifecycle

10. Mount calls the provider's `generateOnce` exactly once and resolves `getResult()` to the returned `{ blob, contentType }` shape.
11. Provider rejection → `onError` fires with `{ kind: 'generate-error' }`; `getResult()` stays `undefined`.
12. The factory creates an `<img>` element under `ctx.root` and sets `img.src` to a blob URL once generation resolves.
13. `regenerate()` aborts any in-flight call, invokes a fresh `generateOnce`, updates `getResult()`, and emits a fresh `generate.started` / `generate.resolved` pair.

### Resource cleanup

14. `MountHandle.dispose()` calls `signal.abort()` on the in-flight call (verified via spy).
15. `MountHandle.dispose()` calls `URL.revokeObjectURL` for any blob URL the factory created (verified via spy).
16. `MountHandle.dispose()` detaches the `<img>` element from the root.
17. `MountHandle.dispose()` is idempotent.

## Escalation triggers (CLAUDE.md §6)

- The host-injected `Generator` callable shape doesn't fit a real-world image-generation SDK (e.g., OpenAI returns `data[].url` not `Blob`; Stable Diffusion returns base64). Escalate; we may need to ship a typed adapter library up front rather than as a Phase-14 follow-up.
- A reviewer pushes back on the "no default real provider" stance (same as T-391's escalation trigger). Escalate per the §3 prohibition on `fetch` inside `clips/**`.
- The blob-URL revocation step interacts badly with React 19's concurrent rendering (e.g., `<img>` mid-paint when the URL is revoked → broken-image flash). Escalate; we may need a paint-cycle delay before revocation.
- The `negativePrompt` field is requested as a `Record<string, string>` (per-aspect negative-prompts, e.g. Midjourney `--no people`). Escalate; the v1 string-only shape is provider-neutral.
- A consumer requests `modality: 'audio' | 'video'` in the v1 schema. Escalate per the deferral; adding now creates a single-value enum with no purpose.
